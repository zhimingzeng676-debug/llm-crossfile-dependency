# TRANSFER_CALLGRAPH.md — 规律迁移验证:调用图任务(小样本 demo)

> **动机(回应评委第三条"深而窄"):** 主线规律("喂确定性算好的结构 > 让模型猜")此前只在**跨文件依赖**这一个任务上验过,泛化是"断言"的。本文在**第二个、份量不同的任务(跨文件调用图)**上做**小样本(n=30)迁移验证**,看规律是否迁移。
> **明确定位:这是小样本 demo 级证据(30 条),非完整研究。** 判分无关(gold = repo_parser 静态调用解析),不挑口径,两种结果都接受。

## 做法
- **任务**:"给定函数 F,它调用了哪些**定义在其它文件**的项目内函数?"——仍在代码结构分析范畴,但与"依赖问答"是不同任务。
- **judge-independent gold**:复用 `repo_parser`(`03_代码/src/repomind_lab/repo_parser.py`)的 **import 解析 + `resolve_symbol` 别名链**,对 werkzeug 算出每个 F 的跨文件被调函数(只取"被 import 进本文件的简单名、解析到另一文件、定义唯一"的干净跨文件信号)。脚本 `build_callgraph.py`。
- **样本**:30 个 caller(覆盖 18 个文件,16 个有 ≥2 跨文件被调、14 个单被调,平均 1.7)。
- **两条件(生成器 Qwen2.5-14B)**:① **baseline** = 只给 F 的源码 + 问题;② **+结构卡片** = F 源码 + 静态分析算好的"F 的跨文件调用目标(name → 定义文件)"。det gold-recall 打分(gold 函数名是否被答出)。

## 结果(det gold-recall,n=30)
| 条件 | recall |
|---|---|
| baseline(只给函数源码) | **0.824** |
| +结构卡片(喂算好的跨文件调用) | **1.000** |
| 提升 | **+0.176(1.2×)** |

## 诚实定论(两种结果都接受 → 落在"方向迁移、幅度收窄"之间)
1. **规律方向迁移**:喂确定性算好的调用结构**确实有帮助**,把 recall 从 0.824 抬到 **1.000(满分)**——"喂结构 > 模型猜"在调用图任务上**方向成立**,规律不止于依赖任务。
2. **但幅度远小于依赖任务,且这是有原因的(关键诚实点)**:调用图 baseline **已高达 0.824**(依赖任务 baseline 仅 ~0.16~0.29)。原因:**"F 调用了谁"这个结构事实,调用名literally 就写在 F 自己的源码里**(`parse_date(...)` 就在函数体中),模型只需从源码读出调用名 + 判断哪些是跨文件,本就做得不错;而依赖任务的"**谁 import 我 / 跨文件符号解析到哪**"在被查文件自身源码里**看不见**,所以那里 baseline 极低、喂结构的增量极大(+0.6)。
3. **精炼后的规律(比原头条更准)**:**"喂确定性算好的结构 > 模型猜"跨任务成立;但其价值大小取决于"该结构事实能否从局部源码看出"**——
   - 看不见的结构事实(反向依赖"谁 import 我"、别名解析、跨文件符号定位)→ 喂结构是**大杠杆**(+0.6);
   - 大体能从局部源码看出的结构事实(正向调用"我调了谁",名字就在源码里)→ 喂结构**仍有帮助但增量小**(+0.18,到满分)。
   这恰好说明**结构喂入在"结构最难从局部看出"的地方最有用**——而那正是工程上最需要的地方(反向 / 跨文件不可见依赖)。
4. **卡片到 1.000 的说明**:结构卡片里直接含算好的跨文件调用目标(= gold),模型读出即满分——这是 RAG"把算好的结构喂进去"的预期行为;**真正信息量在 baseline 0.824**(没有结构时模型自己能到多少)。

## 边界
- **小样本 demo(n=30)**、单生成器(Qwen2.5-14B)、单 temperature;非完整研究,不外推为强结论。
- gold 取"import 解析得到的干净跨文件调用",未覆盖属性调用(`obj.method()`)/动态调用等;是调用图的一个可静态算的子集。
- 结论是"规律方向迁移 + 幅度任务依赖",**不是**"完整泛化到所有代码任务"。

**复现**:`build_callgraph.py`(构数据)+ 远端 `cg_run.sh`(生成)+ `cg_scores.json`(结果);bundle `bundle_cg_{baseline,card}.json`。

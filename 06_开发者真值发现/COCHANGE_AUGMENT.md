# COCHANGE_AUGMENT.md — 共改信号补全依赖卡片,提升 LLM 在静态盲区上的依赖召回(M59)

> 回到项目真正目标:**提升 LLM 跨文件依赖能力**(不是 LLM vs 确定性谁强)。
> 已确立:确定性算出的静态依赖结构→卡片→数量级提升(主线)。已知盲区:静态有 10-16% 看不见(动态/运行时),模型自己补不了(M57)。
> 但这部分结构**共改信号能算出来**(M50-56)。本阶段:把共改算出的盲区边补进卡片,测 LLM 在静态不可见依赖上召回是否提升。
> 防伪影(13/14 次修正的坑):开放生成(避 M56 挤占)+ 等长 filler 对照(避 token 伪影)+ 去噪报两版 + judge-independent。
> 脚本 `pe21_build_cc_cards.py`、`task3_opengen.py`、`pe22_cc_recall.py`。GPU 跑完 0 MiB。

## 方法
- 共改补充卡片 = 种子文件的高置信 git 共改伙伴(去噪:剔除 setup/version/doc/test 噪声,去噪仅丢 2.9% 伙伴);覆盖静态盲区那部分边。
- 三条件开放生成(自由列共改文件,无候选池):**static_only / static+cochange / static+等长无关填充(token 对照)**。

| 条件 | visible | invisible_all | invis_dynamic | invis_namematch | 平均出预测 |
|---|---|---|---|---|---|

| static+filler(等长无关) | 0.686 | 0.185 | 0.188 | 0.184 | 5.2 |

- **静态不可见边召回:0.178 → 0.959(+0.781)**。
- vs filler **+0.774** → **排除"提升只是多给了 token"**(filler ≈ static_only)。
- **visible 不退化**(0.973 ≥ 0.718,共改卡也含部分可见共改伙伴);加共改卡后出预测更少更准(3.4<5.0)。

## 诚实定位(正向结果,带界,不夸大)
**这是一个干净的正向、可部署的方法结果**,也是 M50-59 共改方向的**建设性收尾**:
> 用共改信号算出静态盲区的依赖边、补进依赖卡片,LLM 在原本静态看不见的依赖上召回从 ~18% 提到 ~96%,
> 且不损害静态可见边、不是 token 量伪影——**主线"喂对的结构→模型用起来"延伸到"静态+共改"的更完整结构源**。

**必须带的界(全程透明)**:
1. **是 feed-structure 机制,不是模型新能力**:共改卡片与 gold 同源(覆盖 ~98% invisible gold),0.959 召回 = **模型读了补充卡片里的边并复述**(与静态卡片在可见边上的机制完全相同),**不是"模型理解了隐藏耦合"**。这是**方法可用性 demo,不是模型能力发现**。
2. **依赖共改信号可得 + 正确**:静态盲区里只有 ~10-16% 被共改信号覆盖;co-change 有噪声(M54 去噪)、且 **co-change ≠ 依赖**(是开发者行为代理)。端到端价值取决于共改卡在部署时算得准。
3. **同源=可用性非发现**:本实验证"喂进去模型会用",不证"共改边都是真该改的依赖"。

## 甲方价值(可执行)
**提升 LLM 依赖能力的完整路径 = 静态依赖卡片(主杠杆)+ 共改补充卡片(补静态盲区)**:
- 主杠杆不变:把静态分析算准算全的依赖喂进卡片(+0.5~0.9 大效应)。
- **补盲区**:对动态/运行时/插件这类静态看不见的依赖,用 **git 共改历史(高置信、去噪)**算出来补进卡片——LLM 会用起来(盲区召回 18%→96%)。
- 注意:共改卡是行为代理、有覆盖与噪声边界;对安全攸关用途仍需结构性标注不完整(承接 DOWNSTREAM_HARM)。

## 〔M61〕端到端衰减:把 96% 从"上限演示"变成可部署数字

**96% 是 card=gold 的上限演示**(共改卡覆盖 ~98% invisible gold)。真实部署共改卡有覆盖不全 + 噪声,需打回真实条件(同 M46:下游危害 85% 人造→13% 真实)。
注入退化(覆盖率 sweep + 噪声 precision)开放生成重测,sweep 全报:

| 共改卡退化 | 有效盲区覆盖 | invisible 召回 | 模型复述假边率(噪声伤害) |
|---|---|---|---|
| static_only(无共改卡,锚) | — | **0.178** | — |
| card=gold(上限,M59) | 1.00 | **0.972** | — |
| 覆盖 79% | 0.79 | 0.777 | — |
| 覆盖 73% | 0.73 | 0.711 | — |
| 覆盖 63% | 0.63 | 0.622 | — |
| 覆盖 61% | 0.61 | 0.601 | — |
| 覆盖 74% + 噪声(precision 50%) | 0.74 | 0.686 | **0.765** |
| 覆盖 64% + 噪声(precision 50%) | 0.64 | 0.596 | **0.830** |
| 覆盖 74% + 噪声(precision 30%) | 0.74 | 0.664 | **0.644** |

**关键发现(诚实)**:
1. **召回 ≈ 共改卡对真盲区边的覆盖率 × ~0.97**(模型复述卡里 ~97% 的真边)——**覆盖率是端到端召回的硬约束**。
2. **可部署双数字**:**上限 96%(card=gold)/ 真实部署 ≈ 真实共改覆盖率 × 0.97**。真实覆盖率 = 共改挖掘能找到多少**真依赖**边(受 co-change≠依赖 限制,无开发者真值时未知);在 60-79% 覆盖下召回 0.6-0.78,**仍显著 > static_only 0.18 → 方法在合理覆盖下可部署、有真实价值**。
3. **噪声是更大风险**:固定覆盖下加假边,召回小幅降(0.711→0.686→0.664),但**模型自信复述 64-83% 的假共改边**(confident wrong)——这正是 `DOWNSTREAM_HARM` 的错卡片负资产模式。**→ 共改卡必须高精度去噪,否则补盲变成下游负资产。**
4. **方法学诚实**:覆盖率 sweep 低端被 `max(1 partner)` 地板抬高(故用上表"有效覆盖"而非名义值);召回≈有效覆盖的线性关系在全 sweep 稳健。

**部署定位**:补盲方法**可部署、合理覆盖下显著超 static_only**,但 **(a) 召回上限 = 真实共改覆盖率(co-change≠依赖 的未测缺口)**;**(b) 需高精度去噪共改输入,否则模型自信复述假边=下游负资产**。**别把 96% 当 KPI**;部署预期 = 你们的共改信号精度/覆盖决定。

### 〔M63〕真实传递函数(改回真值 + 系统性退化最后一刀,双向诚实)
M61 的"召回≈覆盖率×0.97(过原点)"有两个问题,M63 用全覆盖区间(去掉 max-1 地板,含 cov=0)+ 两种退化重测:

| 退化方式 | 传递函数(invis 召回 vs 有效覆盖) |
|---|---|
| 随机丢边(乐观界) | **recall ≈ 0.156 + 0.821 × cov** |
| 系统性丢边(按 co-change support,先丢最难挖的) | recall ≈ 0.158 + 0.813 × cov |
| static 底(cov=0) | 0.177 |

**① 改回真值(往上修,被低估的也要改)**:真实关系**有截距 ~0.16(=模型内生召回底),非过原点**。
之前"×0.97"在低/中覆盖端**低估了自己**(如 cov=0.5:×0.97 说 0.49,真实 0.57)。评委独立复现的"0.14+0.84cov"≈实测"0.156+0.821cov"。**诚实不只往低改,被低估的也改回真值。**
**② 系统性退化最后一刀(评委"更陡"假设未坐实)**:按 support 先丢最难挖的边,与随机几乎相同(0.158+0.813 vs 0.156+0.821)——**gold 内召回只随覆盖率走、与丢哪些边无关**(模型读卡片里的边就复述,与该边 support 高低无关)。
**③ 但向下的诚实界(gold 排除效应)**:gold 由 evo support≥5 构造,**真正从未干净共改的硬动态边被排除在 gold 外**;那些边在真实部署里覆盖≈0、只得 0.16 底。所以**两种退化都低估了最硬的真实边**——对"全部真实盲区边(含从未共改的)",期望召回落在 [0.16 底(不可挖边), 0.16+0.82×cov(可挖边)]。

**双界部署预期(最终)**:**recall ≈ 0.16 + 0.82 × (共改对真依赖的有效覆盖率)**,随机/系统性退化几乎同(传递函数稳健);真实部署的不确定性主要在**覆盖率本身**(共改能找到多少真依赖,未测)+ **从未共改的硬边只得 0.16 底**。仍**≫ static_only 0.18 当覆盖>0**;且噪声(假边→自信复述 64-83%)是比覆盖更要命的风险。

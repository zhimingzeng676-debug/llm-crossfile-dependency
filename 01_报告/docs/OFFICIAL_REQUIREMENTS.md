## 项目:基于微调/PE/RAG 的代码分析领域效果优化

### 考核题目(中难度=最高档,要全部覆盖)
1. 【瓶颈诊断】选定一个代码分析方向,在**真实开源项目**上构建 **≥50 条**评测用例,
   评估主流 LLM 基线,诊断低分场景的**共性瓶颈**(长上下文丢失/隐式依赖忽略/
   跨文件关联断裂等)。
2. 【PE 系统优化】System Prompt(角色+输出格式约束)、**Few-shot 示例库(≥20 条)**、
   CoT 推理引导、输出后处理(过滤/排序/去重),**量化每项优化的效果增益**。
3. 【RAG 增强管线】Embedding 选型调优、语义+关键词混合检索、上下文窗口管理、
   检索与生成融合策略,评估 RAG 提升。
4. 【模型微调】**微调数据集 ≥500 条**,LoRA/QLoRA 领域适配,**监控过拟合**,评估效果。
5. 【消融实验】完整消融矩阵:**PE only / RAG only / FT only / PE+RAG / PE+FT / All**,
   识别最优组合及其**适用边界**。

### 难度分档(只有两档,中难度是顶格)
- 低:一个方向 + ≥50 用例 + 纯 PE + 量化前后差异 + PE 文档与对比报告。
- **中(我们的目标):PE 基础上 + RAG Pipeline + ≥500 条微调数据 + LoRA 微调
  + 完整消融,输出各策略独立及组合效果的量化对比报告。**

> 用户补充口径:中难度的硬指标是**及格线不是终点**。每项都要做到超出要求的
> 深度——用例分难度+刁钻隐式/间接依赖、消融带敏感性检查+适用边界、微调认真
> 监控过拟合+误差归因、检索补 Recall@K/MRR。即"在中难度档位上把每项做到
> 经得起答辩追问的质量"。

### 验收评分项(产出逐条对齐)
- **瓶颈诊断质量**:用例基于真实项目/有明确答案/覆盖不同难度;瓶颈识别精准
  (非泛泛);结论有评测数据和错误样本佐证。
- **PE 方案质量**:覆盖 System Prompt/Few-shot/CoT/后处理四维度;每项独立效果数据;
  Prompt 文档详尽到他人可直接复用。
- **RAG 方案质量**:向量索引→检索→上下文管理→融合 各环节完整;检索精度用
  **Recall@K、MRR** 等指标;端到端提升幅度。
- **消融与整体**:单一手段+组合的完整消融矩阵;明确最优组合及适用条件;
  完整代码/配置/脚本/README 可复现。

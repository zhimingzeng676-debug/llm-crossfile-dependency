# GRAPH_REPRESENTATION.md — 本项目"图表示"的真实形态(M18 任务一,代码确认)

> 目的:用**代码事实**(非印象)讲清本项目的依赖"图"到底以什么形态进入 LLM 上下文,
> 为 M19 图表示梯度实验定基线。结论先行:**后台是真图、喂给模型的是图的 1-hop 文本投影**。

## 1. 两层必须分开看:后台图 vs 喂给模型的表示

### 1.1 后台:确实是结构化的异构有向多重图(`repo_parser.DepGraph`)
`src/repomind_lab/repo_parser.py` 用 tree-sitter 解析出一个真正的图对象:
- **带类型的节点**:`FuncInfo`(函数:file/start/end/calls)、`ClassInfo`(类:file/bases)、文件(`graph.files`)。
- **带类型的有向边**(都有显式方法):
  - `import_edges()` 文件→文件(import 依赖);
  - `call_edges()` 函数→函数(调用);
  - 继承边:`ClassInfo.bases` + `resolve_symbol()` **别名解析**(把 `_SansIORequest` 还原成 `Request@sansio/request.py`)。
- **反向边**后台可算:`importers_of` / `called_by` / 子类集合。
- `resolve_symbol(file,name)` 做跨文件**符号消歧**(import 别名、re-export)。
→ 这一层是**异构(3类节点)、有向、多关系(import/call/inherit)、带别名解析**的依赖图,是真图。

### 1.2 喂给模型:图的"每节点 1-hop 邻接表"文本投影(`chunking.load_graph_chunks`)
进 LLM 上下文的**不是**图结构,而是把上面的图**逐节点翻译成自然语言卡片**(`Chunk(kind="graph")`):
- **文件卡片**:`它依赖(import)的仓库内文件: …` / `它被这些文件依赖: …` / `文件内定义的类/函数: …`
- **类卡片**:`它继承自: Request [sansio/request.py]` / `继承它的子类: …`(基类经别名解析)
- **函数卡片**:`它调用的仓库内函数: …` / `它被这些函数调用: …`
每张卡片 = **一个节点 + 它的 1-hop 正向/反向邻居**,序列化成文本。反向边在**建索引时就预先物化**进卡片
(所以"谁 import 了 X"这种反向查询变成一次普通的词面命中,不需查询时遍历)。

## 2. 检索单位:独立文本块,无查询时图遍历
`retrieval.FusionBackend`(`src/repomind_lab/retrieval.py`)按 `chunk.kind` 分槽(`graph` vs `function`/`code`),
用 **BM25 + 语义向量混合召回 + cross-encoder rerank**,把**整张卡片当一个独立文本块**检索。
- **没有子图检索**:检索单位是单节点卡片,不是子图/路径。
- **没有查询时多跳扩展**:不做"取回 A 卡 → 顺边扩到 B 卡"的遍历;多跳(间接/传递依赖)只能靠
  ① 单卡里恰好已含答案,或 ② LLM 在上下文里**自己串联多张卡片**。**传递闭包未预物化(只到 1-hop)。**

## 3. 对照业界图谱类型:本项目落在哪一档
| 维度 | 本项目现状 |
|---|---|
| 后台数据模型 | **异构有向多重图**(typed nodes/edges + 别名解析)✅ 真图 |
| 进模型的表示 | **图的 1-hop 邻接表文本投影**(per-node NL 卡片),非三元组/非 GraphML/非邻接矩阵 |
| 检索粒度 | **节点级卡片**,独立文本块;**无子图检索、无查询时遍历** |
| 多跳能力 | **仅 1-hop 预物化**;传递依赖靠 LLM 在上下文里串联(未结构化支持) |
| 业界对标 | 接近 **GraphRAG 的"节点摘要文本化"**,但更简(只有节点邻接卡,无社区/层次摘要);
|  | **不是**真正的"结构化图喂模型 / KG 查询 / 子图检索 RAG" |

**一句话定档**:**"后台用图算、文本喂模型"的图文本化投影(text-projection GraphRAG, 1-hop, node-level)**——
不是 baseline 误以为的"真结构图 RAG",也不是朴素代码 RAG。这正是它在**反向依赖**上强(反向边预物化)、
在**间接/多跳依赖**上弱(传递闭包未物化)的代码层根因。

## 4. 对 M19 图表示梯度实验的基线定义
当前基线 = **B0:1-hop 节点邻接卡片(文本投影)**。M19 可在干净金标准(见 `DECOUPLED_GOLDSET.md`)上对照的梯度:
- **B1 结构化三元组**:把卡片换成 `(A, imports, B)` 三元组列表,看结构化形式是否优于自然语言投影;
- **B2 多跳预物化**:把传递依赖闭包(2-3 hop)也写进卡片/或查询时遍历扩展,直击当前间接依赖弱点;
- **B3 子图检索**:命中节点后按边扩 1-2 跳取回邻居子图,而非单卡。
**红线**:所有图梯度对照必须在**去循环金标准**上做,否则同源循环会污染图实验(见 DECOUPLED_GOLDSET / E45)。

## 5. M19 梯度实测结果(E46):图形态"按问题所需物化"
在去循环金标准(50 案例)+ 双独立裁判、控制变量只变卡片格式下实测(独立 Coder 裁判):

| 类型 | B0(1-hop) | B1(三元组) | B2(多跳物化) | 结论 |
|---|---|---|---|---|
| forward_dep | 0.924 | 0.920 | 0.921 | 三表示天花板,**形态无所谓** |
| reverse_dep | 0.820 | 0.924 | 0.860 | B1 边际 +0.10(p=.08),非决定性 |
| symbol_location | 1.000 | 1.000 | 1.000 | 天花板 |
| **indirect_dep** | **0.400** | 0.547 | **0.992** | **B2 决定性 +0.592(p=0.001)** |
| 全体(50) | 0.753 | 0.814 | **0.904** | B2 +0.152(p=0.004),全靠间接 |

**核心发现**:**B2 多跳传递物化把间接依赖从 0.40 打到 0.99** —— 直接证实第 1-3 节的机理诊断
(B0 只 1-hop、传递闭包未物化 → 间接依赖弱;物化即把"多跳推理"降为"读取")。
**而直接依赖类型(forward/symbol)三表示全在天花板,图形态无所谓。**
→ 精确结论:**"按问题所需物化"**——1-hop 问题 B0 即可,多跳问题必须 B2;
"选图机制"仅在**多跳轴**上有数据支撑(直接依赖轴收益为零),不是"越花哨越好"。
（注:inheritance 因邻域未含别名解析后的基类卡片,三表示全 0,属上下文构造局限,未纳入结论;internLM 裁判对 indirect 宽松饱和至 1.0、分辨不出,以 Coder 为准。）

## 6. M22:B3 真子图检索 —— 完整图RAG值得上吗(E49)
B3 = 第一次真正的**图结构子图检索**(起点遍历出连通子图:节点+全部内部互联边→序列化),非文本投影。同口径对照(Coder):

| 类型 | B0 | B1 | B2(物化) | **B3(子图)** | B3−B2 |
|---|---|---|---|---|---|
| forward_dep | 0.924 | 0.920 | 0.921 | 0.909 | ~0 |
| reverse_dep | 0.820 | 0.924 | 0.860 | 0.852 | ~0 |
| symbol_location | 1.000 | 1.000 | 1.000 | **0.292** | **−0.708** |
| indirect_dep | 0.400 | 0.547 | **0.992** | 0.828 | **−0.164** |
| 全体 | 0.753 | 0.814 | **0.904** | 0.692 | **−0.212 (p<0.01)** |

**裁定:完整图结构子图检索(B3)对依赖问答是过度工程,不值得上。**
- B3 整体显著低于 B2;**symbol_location 崩**(纯子图丢了节点的"定义符号"属性);
- **indirect:B3>B0 但 <B2**——即便 B3 子图含完整连通边,**模型读 B2 预物化路径比自追溯 B3 子图边更可靠**。

## 7. 图表示选型最终结论(B0–B3 闭环)
| 查询类型 | 最划算表示 | 理由 |
|---|---|---|
| 直接依赖(forward/reverse) | **B0**(1-hop 卡片) | 已天花板,更复杂表示无收益 |
| 符号定位(symbol_location) | **B0**(卡片含节点属性) | 需节点"定义符号"属性,纯图结构会丢 |
| 多跳/间接依赖 | **B2**(文本化物化传递路径) | 把多跳推理降为读取,0.40→0.99 |
| (无) | ~~B3 完整图检索~~ | 无独立价值、会丢属性、不如物化路径 |

**对业界"图 RAG"热潮的冷静注脚**:对依赖问答,**关键是"给什么结构"(把所需路径文本化物化),而非"用多复杂的图检索机制怎么取"**。
轻量文本化物化(B2)已达到/超过完整图结构检索(B3),**完整图数据库 RAG 性价比低,不建议为此任务深做**(见 NEXT_STEPS)。

## 指控二:统计用"伪重复"——把 15 次重跑当样本量 →【成立】

**核实动作**:读 `scripts/analyze_multi.py` 的 `compare()`,并用正确单位(56 用例配对)重算(`reaudit_stats.py`)。

**磁盘证据**:`compare()` 第 63-76 行——`per_run_overall(name)` 返回**每 run 一个 overall(共 15 个)**,
直接 `stats.ttest_ind(a, b)` 在这 15 个 run 均值上做 t 检验。即:
- **t 检验的 n = 15(重跑次数),不是 56(用例)** → 伪重复(pseudo-replication)成立;
- 用的还是**非配对** `ttest_ind`,而两条件共享同一套 56 题,本应**配对** → 双重错误;
- 15 次重跑只是对同一套 56 题的 15 次带噪测量,跨 run σ≈0.012 极小 → 制造出 p≈1e-33 的假精度。

**正确口径(n=56 用例,配对 t 检验)重算结果:**

| 对比 | 老口径 p(15 run, 非配对) | **正确口径 p(56 用例, 配对)** | delta | 裁决 |
|---|---|---|---|---|
| RAG 主效应 full−baseline | 3.3e-33 | **~0(p<1e-10)** | +0.596 | 仍极显著 ✅ |
| **CoT 增益 pe_cot−full** | 1.1e-03 | **0.034** | +0.0197 | 勉强擦边,且见下 |
| CoT 增益(deep 口径) | 2.9e-04 | **0.072** | +0.031 | **不显著** |
| System-prompt | 0.48 | 0.78 | +0.003 | 不显著(本就) |
| 领域 few-shot | 0.22 | 0.47 | +0.006 | 不显著(本就) |
| +FT(all−pe_cot) | 0.76 | 0.93 | +0.002 | 不显著(本就) |

**裁决:指控成立。** 老 p 值(1e-3 ~ 1e-36)是伪重复的产物。

**关于评委的具体数字**:评委称 CoT p 从 0.001 变 0.269。我的正确配对口径得 **p=0.034**(不是 0.269)——
差异在于我**先把每题 15 run 平均**再配对(降噪,df=55),评委可能用单 run。**但方向完全一致:CoT 失去稳健显著性。**
而且即便最有利的 0.034 也站不住,因为它同时被三件事击穿:
- **deep 检索口径下 p=0.072(不显著)**;
- **独立裁判下 −0.024,p=0.29(指控一,不增反降)**;
- **增益 0.0197 < 裁判分级 MAE 0.041(指控三)**,落在裁判分辨率之下。

→ **CoT 增益结论不成立,收回。** RAG 主效应在正确口径下仍极稳健(评委亦承认)。

**修复动作**:
1. 重写 `analyze_multi.py` 的 `compare()`:改为**56 用例配对** `ttest_rel`(每题先跨 run 平均);15 次重跑仅用于降噪,不作 n。
2. 全文凡引用 p≈1e-30 级数字的(CONFIDENCE/EVALUATION_REPORT/EXPERIMENTS/INSIGHT)一律改为 56 用例配对 p,并删除"约 12×σ"这类基于伪重复的表述。
3. 收回所有依赖"CoT 显著"的二级结论。

## 指控三:评测循环 + 裁判校准不足 →【成立】

### 3.1 评测同源(tree-sitter 循环)→【成立】

**磁盘证据**:
- `data/testcases_werkzeug.jsonl` 首行注释:"**答案由静态依赖图导出**;生成器 build_werkzeug_dataset.py"。
  样例金标准 `judge.keywords=["_internal","mixins","structures","exceptions"]` = tree-sitter 抽出的 import 目标。
- RAG 依赖图卡片 = 同一套 tree-sitter import/继承/调用边的文本化。
- `calib_build.py` 第 5 行:人工标注"对照数据集 notes(**来自 tree-sitter 静态分析**)"。
→ **测试金标准、RAG 卡片、裁判参考答案三者同出 tree-sitter,确为同义反复。**

**裁决:成立。** keyword_all 裁判检查的,正是卡片喂给模型的那串 import 名。
这意味着 0.94 部分是"模型能否读回发给它的卡片",而非"系统在真实世界依赖问答上的准确率"——
**RAG 绝对幅度被同源 oracle 抬高了。**

**但不推翻 RAG 主导**:无卡片的 baseline(同一金标准)只有 0.15,有卡片 0.77~0.94。
模型**自己算不出** import 边、必须靠检索拿到——这个 0.6+ 的 gap 真实且有意义。
循环削弱的是"系统解决了任务/94% 真实准确率"的**强主张**,不削弱"提供算出来的结构是数量级杠杆"的**机制主张**。

**修复动作**:把 RAG 结论措辞从"RAG 碾压达 0.94"改为
"**在与构造同源的读回式评测下**,提供 tree-sitter 算出的依赖结构使模型从 0.15→0.94;
真实世界绝对准确率会低于此,正确读法是'模型无法自算依赖、检索结构是必需且数量级主导的杠杆'"。
彻底修复需一个**与 parser 解耦的人工核验金标准子集**(列入重跑计划)。

### 3.2 裁判校准不足(n=20 / 单标注 / 无 IRR / 极端样本)→【成立】

**磁盘证据**(`calib_labels.json` + `calib_build.py`):
- 规模 **n=20**;
- `HUMAN` 字典是**单人(作者本人)**标注,**无第二标注者 → 无 IRR**(结构上不可能算);
- 分布 **16/20 是 0.0 或 1.0 极端**,仅 4 条分级(0.14/0.2/0.22/0.5);
- 实测 MAE:整体 0.022,但**分级子集(4 条)MAE=0.041**(≈2×),且只有 4 个点。

**裁决:成立。** "裁判 MAE 0.022、能分辨 0.02 级差异"的主张**不成立**:0.022 是被 16 条平凡 0/1 样本拉低的;
在真正需要分辨率的分级区间,实测 MAE 0.041 > 任何 0.02 级增益,且样本仅 4。
**裁判撑不起 0.02 级结论**——再次独立印证 CoT/few-shot 级别的微增益不可信。

**修复动作**:① 校准集扩到 ≥60 且**过采样分级分(0.2~0.8)**;② 加**第二独立标注者**算 IRR(Cohen's κ / Krippendorff α);
③ 在新校准给出**分级区间的 MAE 与分辨率下限**,并把"任何 < 分辨率下限的增益不主张"写成硬规则。

## 总裁决与修复计划

**三条指控全部成立。** 受影响结论分两类,必须区别处置:

| 结论 | 受影响? | 处置 |
|---|---|---|
| **RAG(依赖结构)是数量级主导杠杆** | self-judge/伪重复/同源均不推翻 | **保留**(独立 Coder 裁判 +0.578 + 检索指标 + baseline 0.15 三重支撑),但**措辞降级**:同源读回评测、真实绝对值更低 |
| **CoT 在 RAG 之上有小而真增益(+0.02)** | 三条全中 | **收回**:self-judge 伪影(独立裁判 −0.024)、正确口径 p=0.034→deep 0.072→独立 0.29、低于裁判分辨率 |
| System-prompt / 领域 few-shot / FT 冗余 = 无效 | 口径错但结论是"无效",方向不变 | **保留**,p 值改 56 用例配对;FT 比较存在自判/交叉判不对称,加注 caveat |
| 检索深度 rerank 12→80 = 大杠杆(+0.14) | 同 self-judge 口径 | **需独立裁判重验**(并非依赖 CoT 级分辨率,gap 大,预计稳健,但必须重判) |

**需要 GPU 重跑的(干净化,非救火)**:
1. 用**与被评模型不同族的独立裁判**,对 baseline / full / pe_cot / pe_system / pe_domain / deep 全主条件重判 **n=15**,出完全干净数字。
2. 构造**与 tree-sitter 解耦的人工核验金标准子集**(≥20 题,人工查证依赖边),复测 RAG 主效应的"去循环"幅度。
3. 校准集扩容 + 第二标注者 + IRR。

**不需重跑、立即可改的(已用现有数据判定)**:
- `analyze_multi.py` 改 56 用例配对(指控二)。
- 收回 CoT 增益结论;RAG 结论措辞降级(指控一、三.1)。
- 修正 `CONTAMINATION_AUDIT.md`:从"全清白"改为"主结果 self-judge,RAG 经独立裁判稳健、CoT 经独立裁判收回"。
- CONFIDENCE / EVALUATION_REPORT / INSIGHT / EXPERIMENTS / PITCH 同步降级 CoT、降级裁判分辨率主张、加同源 caveat。

> 反思:本项目最强的卖点是"结论可信",而这次外审一次性击穿了三处可信度漏洞——其中两处是我**昨天号称审计干净**的地方。
> 教训不是"审计做了",而是**审计本身也会有逻辑盲区**(self-judge 的判据写错了),以及**越是支撑核心论点的结论越要用独立裁判和正确统计单位去打**。
> 诚实的代价:CoT 这条小增益从此删除;RAG 主导保留但绝对幅度打折。剩下的真结论反而更硬。

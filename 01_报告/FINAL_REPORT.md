## 0. 一页结论(TL;DR)

- **最优方案 = 全栈 RAG(依赖图卡片 + 混合检索 + cross-encoder rerank)+ PE(System
  + CoT + 领域 few-shot)。微调(FT)经六格矩阵正式出局。**
- 真实项目 werkzeug 上,**铁结论是"有无 RAG"这一档**(+0.5~0.7,远超噪声);在其之上 **CoT 再加一个小而真的
  增益**(PE+RAG−RAG only +0.02~0.03,p<0.001,n=15 Welch 确认;跨家族/跨深度复现),而 System-prompt/few-shot/FT 在噪声内。
  **当前最优 = PE+RAG(rerank=80)= 0.94**(M11-12 把欠调的检索深度调对后,werkzeug 0.80→0.94、反向依赖 0.70→0.95;
  新深度下核心结论逐格复查全保持 E38)。第二项目 rich 上规律一致复现 → 方案在所测项目上一致成立。
- **三手段各管一段、不可互替**:RAG 管检索瓶颈、PE/CoT 管推理瓶颈、静态分析卡片
  管符号失联(别名)。**适用边界见第 5 节"瓶颈→解法总表"。**
- **最强单条论证**:把**原始代码**喂给 LLM 做跨文件依赖,分数(0.16)**比不给上下文
  (0.18-0.31)还低** —— 依赖关系不在代码文本里,**必须喂预先算好的结构化依赖卡片**。

## 1. 【瓶颈诊断】真实项目 + ≥50 用例 + 共性瓶颈(对应考核第 1 题)

**项目选型**:werkzeug 3.1.8(52 文件、6 子包),跨文件依赖密度最高
(http.py 被 18 文件依赖、HTTPException 38 类继承群、Request 经别名继承 sansio 子包、
datastructures 包级 re-export 间接链)。选型对比见 DATASET.md。

**用例集**:**56 条**(≥50),6 子类型 × 3 难度,**答案由 tree-sitter 静态依赖图
程序化导出**(可追溯文件/行、可人工核验),刻意埋了别名陷阱/高扇入/多跳/枚举等
hard 靶子。难度标注有效性已验证(baseline 答案分随难度单调下降 0.47/0.24/0.14)。

**baseline + 瓶颈诊断(机理级)**:真实项目 baseline 仅 **答案分 0.27 / Recall@K 0.58
/ MRR 0.44**(mock 玩具数据上是 0.74+,断崖即真实性的证据)。用 **Recall×答案分
交叉定位**,把四类瓶颈精确到机理:

| 瓶颈 | 现象 | 机理 |
|---|---|---|
| **跨文件关联断裂** | reverse_dep Recall 0.77 但答案 0.29 | "谁依赖我"不在文件自身文本里 |
| **隐式依赖忽略-别名** | INH-07 Recall=1 但答案 0 | 基类写作别名 `_SansIORequest`,物理在场符号失联 |
| **隐式依赖忽略-多跳** | indirect Recall 0.40 | A→B→C 中间节点单次检索拿不到 |
| **长上下文丢失** | forward_dep import 埋在大块 | 抽取式捞不出 |

这套"四类瓶颈 + 双指标归因"是后续所有优化的靶子。

## 2. 【PE 系统优化】四维度逐项增益 + 适用边界(对应考核第 2 题)

底座=通用 Qwen2.5-14B + 全栈 RAG,baseline(朴素 prompt)LLM-judge 0.77。
四维度各做"加 vs 不加"独立对照(E21/E26):

| 维度 | Δ | 归因 |
|---|---|---|
| System Prompt(角色+格式约束) | +0.01 | 减少跑题/幻觉 |
| **CoT 推理引导** | **+0.03** | **最有效;间接依赖 0.40→0.70**(多跳推理) |
| Few-shot(通用玩具) | −0.01 | **负**:与 RAG 上下文争注意力 |
| Few-shot(领域真实) | +0.01 | **正**:示例相关才有用 |
| 输出后处理(补全漏列) | +0.01 | 只微帮反向依赖漏列 |

**关键结论**:① **CoT 是 PE 主力**,专治"推理类"瓶颈(间接依赖,n=18 验证 plain 0.44→CoT 0.61,
+0.17 远超噪声 → 稳健);② **few-shot 方向上看"领域真实优于通用玩具"**,但 ±0.01 的幅度
**落在运行方差 σ≈0.014 内,不作分数主张**(回应考核 Few-shot≥20:重点是"用对的示例",
而非刷分);③ **PE 治不了检索瓶颈**——高扇入反向依赖任何 PE 都不动(0.69→0.72)。
**最优 PE = System + CoT(+ 领域 few-shot,作为定性最佳实践而非显著增益)**。
可直接复用的项目无关模板见 `PE_SOLUTION.md`;各结论置信分级见 `CONFIDENCE.md`。

## 3. 【RAG 增强管线】各环节 + Recall@K/MRR + 核心论证(对应考核第 3 题)

**完整 pipeline**:切块(tree-sitter 按函数切)→ Embedding(bge-small-zh,选型见下)→
FAISS 索引 → **混合检索(语义 bge + 词法 BM25,RRF 融合)**→ **cross-encoder rerank
(bge-reranker-base)**→ 上下文组装 → LLM。关键创新是 **依赖图卡片**:把 tree-sitter
解析出的依赖/继承/别名关系**文本化成卡片**进索引。

**Embedding/检索选型(都有消融数据)**:
- 哈希词袋 → bge 语义:history/semantic 方向大涨,但**对精确标识符反而弱**(E1);
- → 混合检索补标识符短板,但**等权 RRF 会稀释强信号**(E3 负结果)→ 权重扫参修复(E8);
- → cross-encoder rerank:词面尺子负优化(E5)、双塔尺子也负(E9)、**交叉编码器才正**
  (E11,MRR 0.52→0.78),这是"rerank 的尺子要带新信息"的三部曲论证。

**Recall@K / MRR(考核要求的标准指标)**:全程作为不经 LLM 的硬信号。werkzeug 上
RAG 把 Recall@5 0.58→0.79、MRR 0.44→0.78。

**RAG 方案的核心论证(本项目最强的一条)**:真 LLM 上,
**原始代码 RAG(0.16)< 纯 LLM 无 RAG(0.18-0.31)**——把代码块喂给模型做跨文件
依赖反而误导它,因为**依赖关系根本不在单个代码块的文本里**。只有喂**结构化的依赖
图卡片**才把分数拉起来(0.16→0.65→0.77,通用模型四档)。**RAG 的价值不在"检索代码",
而在"检索预先算好的结构化依赖知识"**(E18)。

## 4. 【模型微调】数据+隔离+QLoRA+过拟合+诚实裁决(对应考核第 4 题)

**数据(≥500,严格隔离)**:538 条(train 485/val 53),从 flask/click/jinja2/requests
依赖图程序化导出,**werkzeug 整个留作测试、零参与训练**(最强隔离 + 顺带泛化测试)。
重点放难类型间接依赖(142 条)。隔离方式见 DATASET.md。

**QLoRA + 过拟合双防线**:通用 14B,4bit nf4 + LoRA(r=16,可训参数 **0.46%**)。
- 防线一(val 监控):eval loss 0.034→0.0092 单调降不回升,训练分布无过拟合拐点;
- 防线二(项目隔离):防"背测试项目"——这是 val 监控抓不到的。
Model Card 见 `MODEL_CARD.md`。

**诚实裁决(精确,M13续 E40)**:FT 微调生成器对依赖问答**有真实正向效果**——FT only 显著高于无上下文 baseline
(overall +0.043 / hard +0.111,p<1e-4,与文献 FSE2025 RAG-or-FT 一致);但**弱于 RAG 一个数量级**(+0.04 vs +0.72),
且在已有 RAG+CoT 的最优配置上**冗余**(All≈PE+RAG,p=0.50)。**裁决:不进最终方案——理由是"被 RAG 主导且冗余",
不是"FT 无效"。** 训练健康、adapter 加载正常(留出集超 baseline 即证)、无 bug。(M5/导师认可这种诚实分级。)

> **⚠️ M8 置信审计修正(已诚实更新)**:原稿曾据 5 样本宣称"FT 把间接依赖砸了 −0.20
> (0.50→0.30)"。审计一把间接用例扩到 **n=18** 后该结论**未能复现**:ind_ft 0.50 ≈
> ind_plain 0.44,**FT 在间接依赖上与 plain 持平,无崩塌**。原"崩塌"系小样本假象,**已推翻**。
> 同时审计二测得运行方差 **σ≈0.014**,故 FT(0.77)与 PE(0.80)的 ~0.01~0.03 差距落在噪声内,
> **不主张 PE 分数优于 FT**;FT 不进方案的真正理由是"弱于&冗余于 RAG"(E40 精确化),非零增量/性价比为负的旧表述。

## 5. 【消融实验】六格矩阵 + 最优组合 + 瓶颈→解法总表(对应考核第 5 题)

**完整六格矩阵**(werkzeug 56 用例,LLM-judge):

| 格 | LLM-judge | 一句话归因 |
|---|---|---|
| PE only(无RAG) | 0.19 | 无上下文,除间接(参数记忆+推理)全崩 |
| **RAG only** | **0.77** | 检索喂对依赖卡片,单 RAG 就到 0.77 |
| FT only(无RAG) | 0.24 | 学了格式但无上下文 |
| **PE+RAG** | **0.80** | CoT 强化 hard/间接推理 |
| PE+FT(无RAG) | 0.16 | 无 RAG 互相干扰,最差 |
| All(PE+RAG+FT) | 0.79 | 加 FT 无增量(与 PE+RAG 在噪声内) |

**读矩阵**:① **RAG 必要且主导**(无 RAG 三格全 ≤0.24,差距远超噪声 → 铁结论);
② **PE 在 hard/间接上加正**(间接 n=18:plain 0.44→CoT 0.61,+0.17 远超 σ);
③ **FT 不进方案**——FT *单独*正提升(FT only 0.23 > baseline 0.19,+0.04;hard +0.11,p<1e-4,与文献一致),
但在最优 PE+RAG 上**冗余**(All≈PE+RAG)、且**弱于 RAG 一个数量级**;**理由是被 RAG 主导且冗余,非"无效"**(E40)。**最优 = PE+RAG**。

> **⚠️ M8→M9 两轮修正(后者翻回前者一处)**:
> M8 凭 3 轮重跑判"RAG only/PE+RAG/All 三格统计持平,PE+RAG 不优于纯 RAG"。
> **M9 第一层用 15 轮 + Welch t 翻回了其中一条**:**PE+RAG − RAG only = +0.020,p=0.001 显著**
> (CoT 是小而真的信号,M8 系 3 样本假阴性;第三层细粒度裁判显示该增益还被粗判定低估 ~2×)。
> 仍持平的是 **All(+FT) vs PE+RAG**(+0.002,p=0.76)。〔M13续 E40 精确化:旧"FT hard 回归 −0.067"系叠加 PE+RAG + 浅检索;
> 深度修正后仅 −0.009,且 **FT 单独显著提升 hard +0.111**——FT 不伤 hard,只是冗余于 RAG。〕
> 第二层强度梯度另证:"RAG 主导"非"卡片喂答案"伪影(退化卡片分数即雪崩,且 CoT 仅在好卡片下起效)。
> 完整四层归因诊断见 `DIAGNOSIS.md`;分级见 `CONFIDENCE.md` C3/C6/C7。

> **⚠️ M11-12 检索深度修正(本矩阵为 rerank_candidates=12;生产配置已更新为 80)**:
> M11 发现 `rerank_candidates=12` 全程偏浅(漏掉目标依赖卡片),提到 **80** 后所有含 RAG 的格**统一抬升 ~+0.14**:
> RAG only 0.77→**0.90**、PE+RAG 0.80→**0.94**、All 0.79→**0.93**,反向依赖 0.70→**0.95**。
> M12 在新深度下逐格复查(E38):**上表的相对结论全部保持**(RAG 主导、CoT 小而真且略增 0.020→0.031、
> FT 不进最优、PE only<baseline)——**只是绝对天花板被统一抬高,相对关系不是浅检索的伪影**。
> **当前最优 = PE+RAG(rerank=80)= 0.94。** 详见 `EVALUATION_REPORT.md §3a-3c`、`EXPERIMENTS.md E37-E38`。

**泛化验证**:风格迥异、未训练的 rich(终端渲染库),RAG+PE 把 baseline **0.66→0.73**,
**规律一致复现**(CoT 救间接 0.67→1.00、RAG 救反向)→ **方案项目无关、在所测项目上一致成立**。

**瓶颈 → 解法适用边界总表(项目方法论最终形态)**:

| 瓶颈 | 性质 | 正确解法 | 佐证 |
|---|---|---|---|
| 反向依赖(谁依赖我) | 检索 | **RAG**(依赖图反向边卡片) | 反向依赖 0.10→0.69 |
| 别名继承失联 | 静态解析 | **静态分析卡片**(别名还原) | INH-07 0→1.0 |

## 6. 方法论与诚实性(本项目最想被看到的部分)

- **双指标归因**(答案分 × Recall/MRR)贯穿全程,把每个失败定位到"检索的锅 vs
  生成的锅 vs 结构的锅",是所有方案选择的依据。
- **诚实的负结果是骨架,不是污点**:RRF 稀释(E3)、rerank 两连负(E5/E9)、
  通用 few-shot 有害(E21)、微调出局(E24)——每个都归因到机理、保留对照。
- **反过拟合三层防线**:val 监控(防背样本)+ 项目隔离(防背训练项目)+ 风格不同
  的第二项目(防只对某类项目有效)。
- **评测自身先被审视**:LLM-judge 上线前用 20 条人工对照校准(MAE 0.022),并诚实
  修正了 M3 凭直觉下的"keyword 低估真 LLM"的错误说法(实测偏心在 MockLLM 侧)。

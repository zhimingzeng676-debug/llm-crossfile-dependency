- **静态结构覆盖 ~84-90%**;剩 **10-16% 真动态/运行时/插件盲区**静态算不出——靠**共改/运行时信号**算出后喂入(共改补卡 18%→96%)。

## 1. 核心论断(经验证,带边界)

**跨文件依赖分析的瓶颈在"信息可得性",不在"模型推理力"。**

证据链(统计口径:**56 用例配对 t**;关键结论已用**两独立家族裁判**复核,见 `E44`/`M17_RESULTS.md`):
- **〔M29 真实价值边界,最重要的诚实升级〕结构化表示的价值随静态分析准确性翻转符号**:干净(可静态算)依赖 full−baseline **+0.89**;**脏(动态/条件,静态难算)依赖 full−baseline −0.45**(E54,p=0.005)——错/缺的卡片**反而误导**模型(baseline 凭参数知识 0.53 > full 0.08)。**故核心论断不是"定义性碾压",是"价值=f(静态分析准确性, prompt设计)":干净依赖+0.89、脏依赖盲信卡片−0.45、脏依赖humble prompt +0.41(M30 E56-58:翻负稳定+有脏度曲线+但"承认卡片可能不全"的prompt完全救回)。即结构化表示要配"不盲信卡片"的prompt才在脏场景稳健。**M31补完2×2矩阵(humble干净零代价/脏救+0.84);但M33中性措辞复测下修(E60):戏剧性翻负−0.45部分是问题措辞artifact,中性措辞下基本消失、humble救回缩水且不跨裁判稳健。**故这条"脏/humble线"是**证据偏薄的方向性发现**(narrower真实现象:二元determinability问题下错卡片误导),非硬结论(戏剧性端到端幅度是措辞+裁判artifact);**但M34判分无关客观指标证humble实质改善脏依赖识别(strict 21%→humble 100%)是真实的,internLM −0.28是它在脏场景判错**。humble值得用,报实质改善识别非戏剧性涨幅。不与验证32天干净结论同等强度。已知局限三条(人工IRR真空/实际靠单裁判Coder/baseline精确召回格式)见 `HONEST_LIMITATIONS.md`。
- **〔M43 真·新维度:错误依赖卡片的下游伤害——错卡片比无卡片更危险〕** 从"识别准不准"到"误识别下游代价多大"。下游影响分析(改X→谁依赖需查)判分无关实测:**错卡片(漏1依赖方)让模型自信复述不完整影响集、静默漏报隐藏依赖方=89% 危险结局;无卡片模型弃答=0% 危险**(p=4e-16)——**错卡片把"我不知道(安全)"变成"自信却不完整(危险)",是下游危险负资产**;且 **humble 不迁移下游**(错卡片无可见脏线索→审慎不触发,89%→94% ns)。甲方:安全攸关下游绝不能把静态依赖集当"权威完整"呈现、humble 兜不住,要结构性标注不完整 + 危险动作完整性门控。**诚实:真·新维度(主线只管识别、不预测误识别代价),但仍依附"价值=f(静态准确性)"框架,非颠覆主线。** `DOWNSTREAM_HARM.md`/E70。**〔M44 做厚〕** 错卡片>无卡片的危险**跨 4 项目+跨语言(Java gson,wrong 0.77~0.98 vs nocard 0)、跨 3 下游任务(0.80~0.93)、跨错法程度稳健**(E71)。**〔M45 把两边界探扎实——控制实验下都更复杂/更弱,如实报〕**:① **漏报/误报不对称是任务依赖的**(非普适):误报在影响分析=浪费、在安全删除=过度拦截(对真安全删 leaf 100% 误判"不安全"),漏报两任务都危险(E72)。② **humble 检测不出卡片不完整**(控制卡片长度后,humble 对"对/缺"卡片 hedge 一样多,Δ≈0;对正确卡片还 false-alarm 32~54%)——M44"humble 对显眼稀疏卡片审慎"的机理**控制实验下不成立**,**humble 根本不能当下游不完整守卫**(E73)。**净:两个漂亮 M44 边界经控制都被改写,守住了 M41 纪律。** **〔M46 用真实静态分析器漏报验证→诚实下修〕** 评委指 DANGER 0.85 是"随机漏静态边",真实分析器只漏动态/反射。用真实 ast 实际漏报(dynamic14/14、reflection31/31 真漏;**conditional 几乎被 ast 捕获 1/43——条件依赖不是真实静态盲点,纠正早期假设**)在裁剪安全任务上测:真实漏报 DANGER **中口径 0.13(全区间 ~1%~66%,M48 §3h)** vs 完整卡片 0(p=8e-7),**真实危险存在但远低于人造 0.85——人造高估约 6×,如实下修(评委对了)**。还自查出一个被样板免责骗的打分器 bug(会得"DANGER=0 无害"的反向错误结论),改按可执行裁决判定才得真值。E74。
**〔M47 主动证伪一个"太漂亮"的自家假说〕** M46 的 bug 看似揭示一个能统一全局的新机理——"模型言语警觉(说谨慎)与决策行为(给危险裁决)脱钩,humble 只动嘴不动手"。**正因它优雅、能冲优秀,按 M41 纪律往死里验,结果证伪**:P(谨慎措辞|危险决策)=0.98 ≈ P(谨慎措辞|安全决策)=0.99——谨慎措辞是与决策无关的普适样板,"脱钩率"只是算术伪影;且任务依赖、humble 嘴和手都没动、人工核验无干净脱钩案例。**主动撤回,不当新发现。** 唯一保留的朴素真相:模型免责措辞无信息量、不能当安全信号(只看可执行裁决)。E75。
**〔M48 补终审两硬伤 + 措辞降级〕** ① **口径敏感性全报**:下游危害 DANGER 随口径在 **~1%~66%** 漂(中口径 ~18%),我之前单报 13% 取在中-低有利端——把区间公开当结论一部分。② **公平 no_card 对比**:原"错卡片>无卡片"大半是弃答 artifact(no_card 允许弃答时 98% 弃答→0危险);**强制作答下无卡片也危险(中14%/严38%),错卡片只是【略】更危险(中+0.04/严+0.28)**——结论 survive 但削弱。③ **措辞降级(承接终审)**:全项目 judge-independent 真值都是**静态解析器(ast/gcc/javalang)派生、非开发者真实依赖**;严谨表述是"**让模型高保真复现静态分析器输出**"、下游是"**盲信静态分析器输出的后果**",不是"解决依赖理解"。E76/E77。
- **〔M42 主线迁移性验证:碾压不靠记忆,迁移私有代码〕** 把标识符全改名(灭掉"模型背过开源代码"的记忆)后,结构化卡片性能 **100% 保持**(seen 98.6%→unseen 100%,ns);而纯记忆贡献(强制猜)仅 seen 17% vs unseen 2%(改名即灭)。→ **结构(~100%)≫记忆(~15%),且结构对没背过的代码零损失迁移**——RAG 预期干净迁移 RepoMind 私有代码(E69)。**诚实**:这是主线的稳健性**确认**(非新维度),也解决 M41"记忆"张力——模型确有~15%开源依赖记忆,但又小又与卡片机制无关,不威胁主线。`MEMORY_VS_STRUCTURE.md`。
- **RAG 是主导杠杆(大效应,非"数量级"——见顶部 M56 降级)**:baseline 0.19 → 加依赖图卡片 → 调深检索 **0.94**(LLM-judge-independent:Coder 0.915 / internLM 0.954);严打口径约 4-5×、宽松匹配口径约 2×。
  RAG 相对 baseline **+0.64~+0.77,两独立家族裁判 p<1e-18**(self-judge 反而略低估)。
  **去循环验证(M18 E45)**:在与 tree-sitter 完全解耦的金标准(Python ast 独立重抽,与 tree-sitter 100% 吻合 + 人工读源码)上,
  RAG 主效应仍 **+0.66~0.70(p<1e-10)**,循环贡献仅 4~15%——**RAG 主导不是"读回卡片"的评测循环伪影**。
- **模型侧努力一律边际(已穷尽验证)**:**CoT 经三独立裁判全部不显著,已收回**(原 +0.02 系 self-judge+伪重复伪影);System-prompt/few-shot 噪声内;
  **微调生成器**(LoRA +0.04 冗余 / DoRA −0.03 反伤 / IA³ ≈LoRA,E40/E42)、**微调检索器**(对比学习 embedding −0.02 过度专化,E41)、
  **领域继续预训练 DAPT**(无增益,E43)——**换 PEFT 方法、做无监督领域预训练,统统逃不出 0.16~0.23 边际带**。
  真正的杠杆全是**非学习的**:结构(依赖卡片 +0.5)+ 检索深度(rerank 12→80,+0.14)。
  **最深的一击(DAPT)**:让模型大量"阅读"领域代码也学不会依赖——因为依赖是"算"出来的结构事实,不是"读"出来的统计规律。
- **这个结论在六个维度上稳健**:不同检索深度(rerank 12 vs 80)、两独立家族裁判(Coder-14B + internLM,两LLM裁判间一致性 κ=0.806(LLM裁判间,非人工IRR))、
  四个 Python 项目(werkzeug/rich/pydantic/PyYAML)、跨模型家族(通用 14B vs Coder 14B)、**跨编程语言**(Go/Java/C)、**跨规模量级**(中小项目 → CPython 工业级)。
- **工业级规模压力测试(M21 E48):结构化表示在 65× 文件规模(40代表用例验证)下保持 / 14× 卡片池。** 在 CPython(3408 文件)上,
  结构化卡片 vs baseline 强裁判下 **+0.894(p=1.5e-24)**,baseline 近零;**检索没成为瓶颈**(rerank=80 在 1456 池里仍捕获 97.5% 正确卡片,没淹没);
  **可静态抽取的跨语言 Python↔C 扩展边完美(+1.00)**。规模边界=外推到 100×+ 可能需分层检索(未证实);真正边界仍是"结构能否静态计算"。
  **工业级全套六格消融(M23 E50)也整体保持**:RAG 主导更强(+0.89/+0.79)、PE 在强 RAG 之上无增益(−0.002ns,CoT 收回工业级成立)、微调边际被 RAG 主导——小项目的核心相对结论在 CPython 规模逐条复现(一处 PE-only vs baseline 因 baseline 近零而符号翻正,不动决策)。
- **〔M74〕主线跨模型泛化:RAG 主导 / PE 边际不是 Qwen2.5-14B 特性。** 同一 werkzeug 56 + 同一 prompt、仅换生成器,3 模型 / 2 家族 / 2 能力档、**双口径(judge-independent + LLM-judge,0 self-judge)**:**RAG 杠杆 det 2.8×~5.6× / judge 3.3×~5.4×(三模型都大效应);PE(CoT)在 RAG 之上增量 det −0.037~+0.055 / judge −0.024~+0.027(三模型都边际)**。两个诚实新点:① **RAG 杠杆在代码专精 Coder-14B 上最大(det 5.6×)**——它无依赖结构时 baseline 最低(0.166),**连代码模型都得靠喂结构、不能从代码统计里推依赖**,强化"依赖是算出来的结构事实";② **弱档 7B(internlm)上 PE 反而微负(两口径一致)**——证伪"弱模型 PE 更有效",**PE 边际是机理性普适、非没调好**。**边界**:FT 跨模型未验(各自重训成本高,仅 Qwen 验过);Llama-8B 因 host vLLM 兼容失败未纳入(基础设施非挑选)。详见 `CROSS_MODEL_GENERALIZATION.md`。
- **〔强模型验证·界定最致命未控变量〕头号定性"检索受限非容量受限"——经前沿大模型加固。** 评委质疑"模型侧边际可能是 14B 容量伪影"。用 **deepseek-v4-pro**(前沿异族)+ **qwen3-235b**(前沿,比主线 14B 大约 **17×**)在同一 werkzeug 56 + 同一 prompt 双口径(det + LLM-judge glm-5,gen≠judge)重验:**① 前沿 baseline 不高于 14B、甚至更低**(det 前沿 0.21 vs 14B 0.287;judge deepseek 仅 0.04)——**连 ~17× 大的模型都列不出跨文件依赖,容量受限被证伪**;**② RAG 杠杆在前沿上更大**(det 4.6×~4.8× / judge 5.4×~26.8×,full 到 0.98);**③ PE 仍边际**(±0.06,两口径)。**结论:容量不是瓶颈、结构信息缺失才是,头号定性加固而非翻转。** 详见 `07_泛化夯实/STRONG_MODEL_VALIDATION.md`。
- **〔迁移验证·破"深而窄"〕规律在第二个任务(调用图)方向迁移、幅度任务依赖(小样本 demo n=30)。** "给定函数 F 调用了哪些跨文件函数",judge-independent gold=repo_parser 静态调用解析:**喂算好的调用结构 0.824→1.000(+0.18)——"喂结构>模型猜"方向迁移**;但 **baseline 已高达 0.82**(依赖任务仅 ~0.16),因为**"我调了谁"的调用名literally 写在函数源码里、本就能看出**,而依赖任务的"谁 import 我"在局部源码里看不见。**精炼规律:喂结构跨任务有效,但其价值在"结构最难从局部看出的地方"(反向/跨文件不可见依赖)最大**——而那正是工程最需要处。详见 `07_泛化夯实/TRANSFER_CALLGRAPH.md`。
- **跨语言泛化(M20 E47):结构化表示是主导杠杆(大效应,非"数量级"),不是 Python 特性。** 在 Go(gin)/Java(gson)/C(lua)三种依赖机制迥异的语言上,
  强裁判下结构化卡片 vs baseline 全部 **+0.60~0.93,p<1e-6**,baseline 一律 0.05~0.16(模型脱离结构答不出)。
  **连 #include 机制与 Python import 迥异的 C 也照样碾压(+0.86)**。边界=方法依赖"能把结构算出来":
  C 的宏/跨编译单元隐式依赖算不出,就是适用边界(印证"能算出结构才碾压"的前提)。
  **甲方旁证(M26 E52)**:在**腾讯自己的开源项目 rapidjson**(C++)上,结构化表示同样 **+0.83(full 0.996,p=1.1e-12)**——
  最接近甲方场景的旁证(诚实:开源≠私有 RepoMind,但机制语言/项目无关)。产品决策含义见 `VALUE_FOR_REPOMIND.md`。

**为什么是这样**(机理,非玄学):依赖问题的答案是"谁 import/继承/调用谁"这类**结构性事实**,
它**不在任何单个代码块的文本里**——我们实测过:直接喂原始代码 RAG(0.16)比纯 LLM(0.31)还差,
因为代码文本里没有"被谁依赖"的信息。所以必须用静态分析把图谱算出来、文本化成卡片、检索进上下文。
一旦正确结构进了上下文,剩下的推理很轻(读结构 + follow 指令),模型侧技巧自然只剩边际空间。

**边界(诚实)**:此论断限定在**自然语言依赖问答**任务。对**代码补全**等"推理/生成受限"的任务,
结论很可能相反(见 §3)。"检索受限"是任务性质,不是放之四海皆准。

## 2. 三类瓶颈 → 三种解法(适用边界表)

把所有失分归因到三个正交瓶颈,每个有明确归属和解法——这是方案成熟度的核心:

| 瓶颈 | 本质 | 解法 | 效果 | 适用边界 |
|---|---|---|---|---|
| **跨文件关联断裂**(正/反向依赖) | 检索:答案不在单块文本 | RAG 依赖图卡片 + **足够深的 rerank(80)** | baseline 0.19→0.94;反向依赖 0.70→0.95 | 治本;深度是关键超参 |
| **符号失联**(别名/跨文件继承) | 结构解析:`_SansIORequest` 切断符号链 | tree-sitter 别名预解析,卡片里还原真实类 | INH-07/08 0→1.0 | 治本;需静态分析支撑 |
| **隐式/多跳推理**(间接依赖) | 检索:1-hop 卡片未物化传递闭包 | **B2 多跳传递物化**(显式写出 A→B→C 路径) | 间接依赖 0.40→**0.99**(+0.59,p=0.001,独立裁判,M19 E46) | **治本=物化传递闭包**;非模型推理问题(CoT 已收回),仍是"检索/表示给什么"的问题 |
| 高扇入反向依赖 | 曾以为"检索覆盖未解" | **实为检索深度不足**(浅 rerank 漏目标卡片) | 0.70→0.95(调深即解) | M11 修正:不是模型的锅 |

关键洞察:**CoT 在独立裁判下连"小放大器"都算不上(M17 收回)——PE 的唯一确定作用是脱离可靠上下文时反而抑制模型**
(PE only 0.11 < baseline 0.19)。即模型侧的提示工程在这个**检索受限**任务上既不能替代检索,也无可测正增益;一切看检索。

## 3. 与 2026 SOTA 的对标定位

引文献是为了**找盲区、定边界**,不是为结论背书。本项目与前沿的一致与分歧:

| 文献 | 它说什么 | 本项目 | 一致 / 独立发现 |
|---|---|---|---|
| PE 上限(arXiv 2602.13890) | 实证 PE 在代码任务上限 ~6% | 本项目 PE/CoT 经独立裁判**无可测正增益**(M17 收回) | ✅ **更强一致**:PE 在此任务上限不是个位数,而是**统计上为零**——检索受限任务里提示工程几无空间 |

| 选择性检索/自评估(Repoformer,arXiv 2403.10059) | 模型自评估何时需要检索,避免无谓/有害检索 | 本项目做答前自评估 + 选择性 abstain | ✅ **同思路**,并独立量化了它的**可信度价值**(见 §4) |
| 微调检索器(RLCoder,ICSE2025) | RL训检索器(ppl作reward)跨语言超SOTA | 本项目实测两条学习路径**都≤静态分析**:对比学习端到端 −0.02(E41)、**RL/RLCoder Recall 0.958<静态 1.000(E51)** | ⚠️ **本任务上不一致且已穷尽**:RLCoder 的 ppl-reward 85% 指向静态算出的答案→RL **收敛向静态分析、不超越**;依赖是精确可算的结构,静态分析是 oracle,学习去"猜"打不过 |

**本项目的独立贡献**:① 实证"原始代码 RAG < 纯 LLM"——naive RAG 对依赖任务有害,必须结构化;

## 4. 一个有研究品味的独立观点:可信度是独立于准确率的一等评估维度

主流代码工具评测只看准确率。本项目的自评估实验给出一个反直觉但重要的论点:

> **对依赖类代码工具,"自信地答错"比"诚实地说不知道"危害大得多——因为前者会误导开发者去改错地方。
> 因此幻觉率(自信错误率)应当与准确率并列,作为一等评估指标。**

数据支撑:上下文不足时,普通 prompt 瞎猜 → **79% 自信错误**;加自评估(答前判上下文是否足够,不足则 abstain)
→ **0.1% 自信错误**(99.8% 诚实说"信息不足")。代价:好上下文下过度 abstain,准确率 −0.044。

一个让**总分下降**的机制可能是好机制——前提是你把"省下的幻觉"和"损失的覆盖"两条线都摆出来,
让使用场景决定取舍,而不是默认"涨分=好"。**只报准确率,会把一个提升可信度的机制误判成"变差了"。**

## 5. 诚实的研究历程:让数据反复推翻自己

本项目最强的不是某个 0.94,而是**一套敢于自我证伪的方法论**。**十次**有据的自我修正(第 7 次起由外部评委触发;第 10 次=M41 亲手撤掉自己 M38-40 的"新发现",最严重——详见本节末两段 M38-40/M41):

1. **"FT 把间接依赖砸到 0.20"→ 推翻**:原结论基于 5 个样本。扩到 18 个后崩塌未复现,是小样本假象(M8)。
2. **"CoT 增益在噪声内"→ 翻回 →〔M17 再翻回:收回〕**:M8 凭 3 轮误判"噪声内";M9 用 15 轮 Welch t 翻回为"小而真(+0.02 p=0.001)"——
   但 M17 查出那个 p 同时栽在 **self-judge + 伪重复**两个坑上:换三个独立裁判(含两不同家族)+ 56 用例配对后,**CoT 增益全部不显著**
   (Coder −0.034 / internLM −0.005 / deep +0.03 ns)。**最终收回:CoT 不是可主张的正信号。**(详见下 #7 与 `E44`)
3. **"相似度检索是被轻视的更大杠杆"→ 证伪**:实测它单用远差于依赖检索,文献结论不迁移(M11)。
4. **"显式融合 3+2 是新最优"→ 当天被自己的对照组否掉**:deep-mixed 控制显示真杠杆是**检索深度**而非融合,
   reranker 给够深的池子就自适应了(M11)。
5. **"FT 性价比为负/伤 hard"→ 口径精确化**:与文献(FSE2025 RAG-or-FT)对照后做口径核对,发现 FT only 其实
   **显著高于** baseline(+0.04 overall / +0.11 hard,p<1e-4)——FT 是正提升,之前的"伤 hard −0.067"是
   "叠加到 PE+RAG 时"且在浅检索下(深度修正后仅 −0.009)。**改判:FT 正提升但弱于&冗余于 RAG,非"本质无效"**(M13续 E40)。

6. **〔评测污染当场抓出并修复〕**:M15 评测被多实例并发(wakeup 重复启动 detached 编排 → 抢 GPU → 0.0000 失败)
   + self-judge(merged 模型自评把 DoRA 虚高到 0.40)双重污染。当场识别、隔离污染文件留证、单实例锁 + 独立 base 裁判重跑
   (DoRA 真实 0.160),并主动做全历史污染审计(`CONTAMINATION_AUDIT.md`)。**但那次审计有逻辑盲区**(只验"裁判是不是 base",
   没验"被评的是不是同一个 base")——埋下第 7 次修正的伏笔。

7. **〔外部评委触发 · 最重〕self-judge 漏判被外审击穿 → CoT 收回、RAG 反被证实更硬**:外部评委指出 werkzeug/rich **主结果**
   gen=judge=同一个 Qwen-14B = self-judge,而 M15 审计标成了"独立 base 裁判"。核实属实。修复(M17,`E44`/`M17_RESULTS.md`):
   **生成固定、三裁判重判**(self对照 + 独立 Coder + 独立 internLM,两不同家族)+ deep n=15 重生成。结果:**CoT 三裁判全不显著 → 收回**;
   **RAG 主效应三裁判全 p<1e-10、deep 0.94 头条独立裁判下成立(0.915/0.954)、self-judge 反而略低估 RAG**;两独立家族 两LLM裁判间一致性 **κ=0.806**(注:LLM裁判间,非人工标注者IRR)。
   **教训**:审计本身也会有盲区;越支撑核心论点的结论越要用独立裁判 + 正确统计单位去打。这是项目唯一一次**外部触发**的自我修正——
   证明诚实链对外部审查同样成立。

还有一次**预判落空也照实记**:M12 预判"检索更全 → CoT 边际更小",实测反而略增——但该"增益"本身已于 M17 随 CoT 一并收回,以数据为准。

**〔M36-37 · 翻"墙一"〕把核心结论从单一裁判上彻底卸下**:真外部评委指出"卖点是可信,可信度本身却是最大未补洞——压在一个有偏好的 Coder + n=20 单标注 + 人工 IRR 真空上"。
回应不是再补裁判,而是**让最重的结论不需要裁判**:用零 LLM 确定性指标(真值由独立 `ast` 生成、与 RAG 卡片解耦 87%)复算——
**数量级 RAG:det gold-recall 0.159→0.747,paired t=8.68,p=6.9e-12;humble 脏识别:strict 20%→humble 93%(条件依赖 1%→91%)**——一个字符串匹配器都能复现。
**两处诚实下修**:① 确定性 RAG 命中 0.71-0.75 < 裁判 0.94 → 裁判宽松约 0.2,绝对天花板有 ~0.2 是裁判宽松度(硬地板 0.71);② 加固双指标下 humble 只认脏**性质**不定位脏**机制**(条件依赖 I2 仅 1%)。
校准集 20→224(gen≠judge,ρ=0.894/κ=0.795)且撤回同源 κ=0.941 改口径。**仍承认做不到**:真·第二人类标注者(自主 agent 无法)、主观质量维度仍需裁判。`JUDGE_INDEPENDENT_VALIDATION.md`。

**〔M38-40 · 翻"墙二"〕做硬识破维度(保留):** 脏用例 19→**103**、单项目→4 语言 7 项目、反射 n=2→31,判分无关下 **humble 识破脏依赖 strict 7%→99%(p=4.5e-65)**——识破维度是硬结论(detect 低门槛,见下订正)。

**〔M41 · 第10次自我修正,最严重的一次:把 M38-40 的"新发现"亲手撤掉〕** 第六轮评委对墙二"识破≠补全/信息可得性/喂源码90%"两条指控,从零彻查**全部成立**,且暴露第三个更糟问题:
**① 挑口径**:头条 10%→90% 是 {humble|source}×{全样本|determinable} 四格里最戏剧一格(全样本诚实数字 42%→84.5%);
**② 同源循环**:58/58 determinable 片段字面含答案,去循环(答案遮蔽 GPU 重跑)后 90%→**76%**(+14pp 循环未披露);
**③ 机理被自证推翻**:humble_prompt 只问不喂源码也补 **74%**——模型**靠记忆公开库条件导入**补出,"信息可得性"是错的(信息在权重里),且这 74% 是**开源测试集记忆、不迁移私有代码**。
**这是冲"优秀"压力下,在新脏线上重犯了第一轮被外审抓的同两个毛病(挑有利口径 + 同源循环),还多加一个被自己去循环数据推翻的机理过度解读。** 评委对了。**作废这条"新发现",降级为"humble 识破普遍但不主动列具体名、一问靠记忆召回"的方向性观察;承认它本质是项目主线复述且大半是开源记忆,不该当冲优秀的"新"。** `DIRTY_MECH_RETRACTION.md`/E68订正。**目标是真值不是优秀——这条新发现撤了。**

> 这些不是项目的弱点,是它**可信度的来源**。一个从不推翻自己的研究,要么问题太简单,要么没在认真看数据。

## 6. 〔M50-54〕顶破"解析器真值"死结:co-change 负杠杆发现(带界)

**死结**(第八/九轮真外部评委锁死):本项目所有 judge-independent 真值都是静态解析器派生,从未对**开发者真值**验证——
故严格说证的是"高保真复现解析器输出",外推到"解决依赖理解"是未验证假设。评委判:补这个缺口"当前条件给不了"。

**顶破**:用 git **co-change**(开发者修一个真实 bug / 历史共提交时实际一起改的文件)作真值——
它由 git 历史派生、**真正独立于 ast/tree-sitter**(非 parser-vs-parser)。在此真值上得到一个反直觉发现:

> **在静态分析(import + resolve_symbol 精确调用 + 继承)看不见的开发者变更耦合边上,
> 提供静态依赖卡片会显著压低模型对这些边的召回(负杠杆),同时抬高静态可见边召回。**
> no_card−static_card 在 invisible 层 +0.05~0.11,McNemar p<1e-5,跨 Coder-14B/Qwen-14B/internlm-7b 三模型一致。

> ⚠⚠ **M56 机制大幅下修(第 12 次诚实修正)**:这条负杠杆的**机制**经干净对照(random_in_pool:随机文件强制入池,入池率与 static 可比)证实**主要是固定输出预算下的"机械挤占",非"内容特异锚定"**——
> random_in_pool 在 invisible 层压低召回与正确静态卡片**一样多**(0.282≈0.285,**McNemar p=0.94 无差异**),总召回甚至更低(0.344<0.375)。
> M52 的"内容特异"是 random_card 出池(入池率 0.09)的混淆产物。**故本发现下修为:"在固定输出选择任务里,给模型任何入池候选列表都会挤占其对未列出的静态不可见 co-change 边的召回"——比"静态表示认知性带偏模型"朴素得多。**
> 仍真实:co-change 有静态看不见的边、模型凭语义召回约 1/3;给卡片后这些边召回降。**被推翻**:降低是"内容特异"。详见 `M56_RESULTS.md`。

> ⚠⚠ **M57 再下修(第 13 次诚实修正)——"模型语义补盲"也关掉**:用**开放生成**(无候选池,绕开挤占)测"模型召回 1/3 静态看不见边"是不是真能力:召回**存活**(非挤占伪影,gate 通过),
> **但信号分解显示 ~94% 是平凡信号**——text_clue 63%(模型读到种子代码文本里已有的模块名)+ 文件名 10% + 包结构启发(__init__/同目录)21%;**"真语义直觉补盲"残差 ≤1.8%(14 条)且 inspection 显示是知名项目结构记忆,非运行时耦合直觉**。
> **故"模型能补静态盲区"这条线诚实关掉**:模型没有有意义的语义补盲能力,1/3 召回大部分是把种子代码里已有的名字/相似文件名/同包结构重新说出来。详见 `M57_RESULTS.md`。
>
> **co-change 方向去掉两个被下修的漂亮解读后,诚实最终贡献**:① 用正交(非解析器)开发者真值验证主线 + 顶破 parser-truth 死结的方法论;② 量化静态盲区(~10-16% 真动态/运行时);③ 两次诚实证伪("内容特异负杠杆"=机械挤占、"模型语义补盲"=平凡信号)。**价值在方法论 + 量化 + 诚实负结果,不是"补盲能力"或"认知带偏"。**

> ⚠ **M58(第 14 次修正)·角色转换尝试仍回到主线**:让模型当"静态盲区预警器"(二分类"有无静态会漏的动态依赖")——模型非纯关键词(超关键词召回 0.64)、非记忆(改名存活),
> **但过度预警(FP 0.29)、F1 0.755 < 窄关键词基线 0.82**;广义确定性 AST 检测器会主导它。**→ 盲区预警也是"确定性检测器 > LLM",角色转换没逃出主线规律。** `BLINDSPOT_WARNING.md`。

> ✅ **M59 · 共改方向的建设性正向收尾(回到"提升 LLM 能力"真目标)**:把共改信号算出的静态盲区边补进依赖卡片,**LLM 在静态不可见依赖上召回 18%→96%**(filler 对照证非 token 伪影、visible 不退化)。**主线"喂对结构→模型用起来"延伸到"静态+共改"更完整结构源**。
> ⚠ **M61 · 把 96% 打回可部署数字**:96% 是 card=gold 上限(共改卡覆盖 98% gold)。**别把 96% 当 KPI**;噪声是更大风险——假共改边模型自信复述 64-83%(下游负资产),共改卡必须高精度去噪。
> ⚖ **M63 · 真实传递函数(双向诚实)**:① **改回真值(往上修)**——之前"×0.97 过原点"在低/中覆盖端低估自己;真实 **recall≈0.16+0.82×cov(有截距=模型内生底)**,评委复现"0.14+0.84cov"吻合。② **系统性退化(最后一刀)**:按 co-change support 先丢最难挖的边,与随机几乎相同(传递函数稳健,gold 内召回只随覆盖率)。③ **诚实向下界**:gold 由 support≥5 构造、排除了从未共改的硬动态边→两种退化都低估最硬真实边(那些只得 0.16 底)。`COCHANGE_AUGMENT.md`。

> **M50-59 共改方向净账**:① 方法论(正交真值顶破 parser-truth 死结)+ ② 量化盲区(10-16%)+ ③ 两个漂亮解读诚实证伪(负杠杆=挤占、补盲=平凡信号)+ ④ **一个干净正向可部署方法(共改补卡提升盲区召回)**。诚实负结果与建设性正结果并存。

判定为真、并据此把项目从「良好(高位)」上修到「优秀(下限),已达到」的发现。** 不带"会塌"警告(已是真实小幅、真实数据、复现稳固)。

**严格带界(不夸大)**:
1. **co-change ≠ 依赖**:gold 是开发者行为信号,非主观认定的依赖。措辞统一"变更耦合"。
2. **行为层确证,非注意力层证明**:机制("正确静态内容把模型锚定到静态邻居")由 filler/random 对照在行为层坐实,未做注意力级证明。
3. **效应小而极稳**:+0.05~0.11;选择式召回任务、单 temperature。
4. **〔M54·evo 特异〕**:去噪后被压低的边 **97% 是核心源码耦合(非 CHANGELOG/版本噪声)**,负杠杆在 evo 去噪 core_both 子集存活(p=9.5e-6);
   **但在 bug-fix(最强真任务耦合 gold)上是真 null(Δ=0)**——负杠杆特异于 evo 高频历史共改、需"模型本有可观卡片外召回"才显现,**不泛化到单次真任务耦合**。即"压的是真核心耦合非噪声"坐实,但"co-change=真依赖"未跨过。

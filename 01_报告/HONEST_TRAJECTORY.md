## 一、十次有据的自我修正(第 7 次起由外部评委触发)

1. **"FT 把间接依赖砸到 0.20"→ 推翻**:基于 5 样本,扩到 18 后崩塌未复现,小样本假象(M8)。
2. **"CoT 增益小而真(+0.02,p=0.001)"→ M17 收回**:那个 p 同时栽在 self-judge + 伪重复两个坑;三独立裁判 + 56 用例配对后全不显著。
3. **"相似度检索是更大杠杆"→ 证伪**:实测单用远差于依赖检索,文献结论不迁移本任务(M11)。
4. **"显式融合 3+2 是新最优"→ 当天被自己对照组否掉**:真杠杆是检索深度,reranker 给够深就自适应(M11)。
5. **"FT 性价比为负/伤 hard"→ 口径精确化改判**:FT only 实为正提升(+0.04/+0.11),之前是浅检索 + 叠加 artifact(M13续)。
6. **评测污染当场抓出**:多实例并发抢 GPU + self-judge 双重污染,隔离留证 + 单实例锁 + 独立裁判重跑 + 全历史污染审计(M15);但该审计有盲区,埋下第 7 次伏笔。
7. **〔外部评委触发·最重〕self-judge 漏判被外审击穿 → CoT 收回、RAG 反被证实更硬**:主结果 gen=judge=同一 Qwen,核实属实;三裁判重判:CoT 全不显著收回,RAG 主效应三裁判 p<1e-10、self-judge 反而低估 RAG(M17)。
8. **〔M32 自评伪优秀作废〕**:自评"优秀"被后续真评委 + 自查推翻,回落。
9. **〔M41 最严重·亲手撤掉 M38-40"新发现"〕** 冲优秀压力下重犯第一轮被抓的两毛病——挑有利口径(10%→90% 实为全样本 42%→84.5%)+ 同源循环(58/58 含答案,去循环后 90%→76%),且机理被自证推翻(不喂源码也补 74%=靠记忆开源代码)。第六轮评委指出,从零彻查全成立,**作废该"新发现"**(`DIRTY_MECH_RETRACTION.md`)。
10. **〔M46-47 主动证伪自家漂亮假说〕** M46 用真实静态分析器把下游危害招牌数字从人造 85% **诚实下修 6× 到 ~13%**;M47 钻"言行脱钩"新机理,**往死里验后证伪**(P(谨慎|危险)0.98≈P(谨慎|安全)0.99=无信息量样板),**主动撤回不当新发现**。

11. **〔M56·冷启动评委戳破招牌注水〕** 一个完全冷启动、自写代码从原始数据复现的评委,确认两头条可独立复现后,
   戳中**项目唯一一直没撤的两处招牌措辞**:① "数量级主导"——宽松匹配口径下 RAG 杠杆只 ~2×(0.39→0.83),"数量级"被证伪,降级为"主导杠杆(大效应:严打 4-5×/宽松 2×)";
   ② "judge-independent"——只脱离 LLM 裁判、未脱离解析器(gold/卡片/锚同源 tree-sitter),降级为"LLM-judge-independent(parser-anchored)"。
   **盲区自查**:之前十次修正撤的都是"非招牌"项,**对自己最得意的两句招牌反而有盲区**——直到冷启动外部视角戳破。如实降级、全项目订正(banner 级)。

12. **〔M56·机制对照补全,自己推翻自己的"内容特异"解读 + 部分推翻第十轮升级理由〕**
   冷启动评委指出 M52 机制实验的 random_card 对照有混淆(随机文件入池率仅 0.09、物理上选不出→"random≈no_card"不成立)。
   补干净对照 **random_in_pool**(随机文件强制入池,入池率 1.0):它在 invisible 层压低召回**与正确静态卡片一样多**(0.282≈0.285,**McNemar p=0.94**),总召回甚至更低。
   → **M52"内容特异负杠杆"被推翻**:负杠杆主要是**固定输出预算下的机械挤占**(任何入池列表都挤占未列 gold),非"正确静态内容特异锚定"。
   且**第十轮评委用"static 出更多预测却总召回更低"推断的"非预算重分配/内容让模型更差",被 random_in_pool 复现并超过该模式所推翻**——其升「优秀(下限)」的部分基础(内容特异、非机械挤占)被削弱。
   **如实下修发现 novelty,不护;是否仍配优秀交评委重裁。** 这次是"自己补的对照推翻自己上一个机制解读 + 部分推翻了给自己升级的评委的推断"——诚实链对"对自己有利的裁定"同样执行。

13. **〔M57·用开放生成关掉"模型语义补盲"这条线〕** M56 推翻"内容特异负杠杆"后,剩"模型能召回 1/3 静态看不见边"未查。
   用**开放生成**(无候选池,绕开挤占)测:召回**存活**(非挤占伪影),**但信号分解显示 ~94% 是平凡信号**(text_clue 63%=读到代码里已有的名字 + 文件名 10% + 包结构启发 21%),**真语义直觉残差 ≤1.8% 且像项目结构记忆**。
   → **"模型能语义补静态盲区"诚实关掉**。至此 co-change 方向的两个框架外"漂亮解读"(负杠杆、补盲)**都经自己补的干净实验下修为平凡/证伪**;留下的是干净的方法论(正交真值)+ 量化盲区(10-16%)+ 两次诚实负结果。

14. **〔M58·角色转换"盲区预警器"的创新尝试,实证仍回到主线〕** 换结构:不让模型补依赖,让它当**元层预警器**(二分类"有没有静态分析会漏的动态依赖")。
   judge-independent gold + 必报 FP + 超关键词基线才算 + 改名验记忆。结果:模型 **非纯关键词**(超关键词召回 0.64、clean 机制 0.82)、**非记忆**(改名存活),
   **但过度预警(FP 0.29)、F1 0.755 < 窄关键词基线 0.82——未超过简单基线**;且一个广义确定性 AST 检测器会主导它。
   → **第 14 次修正:角色转换没逃出主线规律——盲区预警也是确定性检测器 > LLM**。是干净的负-leaning 工程结论(用确定性检测器做预警),非"模型当预警器很行"。`BLINDSPOT_WARNING.md`。

> 教训贯穿:越想要"优秀/新发现"的结论,越要先用**全样本 + 控口径 + 去循环 + 独立裁判 + 干净对照(入池率可比)+ 开放生成绕开任务形态混淆 + 必报 FP + 超基线才算**自己打一遍再写。冲优秀的压力本身是诚实性的风险源。
> **第 11-13 次的额外教训**:对自己反复验过的"招牌/漂亮机制/漂亮能力"最容易失去警觉——冷启动外部复现 + 补未做的干净对照 + 换任务形态,是戳破自我盲区的不可替代手段;**连给自己升级的评委的推断、自己最想保的"框架外发现",有新证据都要如实下修。最终留下的是干净的方法论 + 诚实负结果,而非戏剧性。**

## 三、M50-54 开发者真值发现:双面如实记

**死结**(第八/九轮锁死):所有 judge-independent 真值都是静态解析器(ast/tree-sitter)派生,从未对开发者真值验证 → 证的是"高保真复现解析器输出"非"解决依赖理解"。

**顶破**(M50-52):用 git **co-change**(开发者行为,真正独立于解析器)作真值,发现:
**静态看不见的变更耦合上,正确静态依赖卡片反而压低模型召回(内容特异负杠杆),呼应 M29。** 扛过 5 道对抗检验(文件名巧合/候选池伪影/上下文挤占 filler/"有列表即锚定"random/跨 3 模型 2 家族)+ McNemar p<1e-5 + 评委第六道攻击。

**M54 双面结果(挡下一刀 + 诚实暴露一界,两面都记)**:
- ✅ **挡下"也许全是噪声"那刀**:被压低的边 **97% 是核心源码耦合、仅 3% 噪声**;负杠杆在 evo 去噪 core_both 子集存活(**p=9.5e-6**)、在 noise 子集消失(Δ−0.033 ns)→ 压的是真核心耦合非噪声。
- ⚠ **诚实暴露 evo 特异边界**:在 **bug-fix(最强真任务耦合 gold)上负杠杆 Δ=0**(真 null)→ 它特异于 evo 高频历史共改、需"模型本有可观卡片外召回"才显现,**不泛化到单次真任务耦合**;"co-change=真依赖"未跨过。

**净**:发现真实性铁、重要性部分坐实(非噪声)但有界(evo 特异);效应小;故维持优秀(下限),未自抬。

**〔M56-59 后续:两个漂亮解读诚实证伪 + 一个干净正向方法〕**(不只记负面,正负并记):
- **M56(第 12 次修正)**:random_in_pool 干净对照推翻"内容特异负杠杆"——主要是选择式机械挤占(p=0.94 无差异)。
- **M57(第 13 次修正)**:开放生成 + 信号分解推翻"模型语义补盲"——开放召回真实但 ~94% 平凡(text_clue/文件名/包结构),真直觉残差 ≤1.8% 且像记忆。
- **M58(第 14 次修正)**:模型当盲区预警器仍输给确定性 AST 检测器(过度预警 FP 0.29)——角色转换没逃出"确定性 > 模型"主线。
- **M59(正向,非修正)**:**共改补充卡片把 LLM 静态盲区依赖召回 18%→96%**(非 token 伪影、visible 不退化)——主线"喂对结构→提升"延伸到"静态+共改"。**诚实带界**:feed-structure 机制(共改卡≈gold 同源),非模型新能力;可部署方法 demo。
- **M61(把正向方法的 96% 打回可部署数字,同 M46)**:M59 的 96% 是 card=gold 上限演示;注入退化后召回随覆盖率降;噪声是更大风险(假边模型自信复述 64-83%)。**主动给自己的正向结果补"理想→真实"打回。**
- **M63(双向诚实:把低估自己的保守简化改回真值 + 系统性退化最后一刀)**:① M61 的"×0.97 过原点"**低估了自己**——真实 **recall≈0.16+0.82×cov(有截距=模型内生底)**,评委复现"0.14+0.84cov"吻合;**诚实不只往低改,被低估的也改回真值(往上修)**。② 系统性退化(按 support 先丢最难挖的边)与随机几乎相同(传递函数稳健);③ 但 gold 排除了从未共改的硬边→两种退化都低估最硬真实边(向下界)。**这是少有的"往上修自己"的修正——诚实是双向的。**
- **总结**:共改方向 6 个想找"模型优势/框架外新发现"的解读逐个证伪/下修,**但最后回到主线做成一个干净正向方法,并主动把它的理想数字打回部署预期**——这正是诚实链的价值:**不护漂亮解读、也不让漂亮正向数字误导,最终落到服务"提升 LLM 能力"真目标的、带诚实部署区间的可部署结果。**

## 三B、把"六次证伪"重定位为"无效提升路径的系统排除"(M60 真目标框架)

## 三C、〔M67〕第 15 次修正:核查"选错语言 embedding"对历史结论的影响范围(地基自查)
M65 发现项目一直用 `bge-small-zh`(中文)做语义 embedding,任务却是英文/代码(语义层 R@5 0.32,换 en→0.93)。**主动核查它影响了哪些历史结论**(不因"主线大概没事"跳过):
- **全实验按 embedding 角色分 A/B**(`EMBEDDING_IMPACT_AUDIT.md`):A 类(embedding 是主角:相似度检索/原始代码 RAG/embedding 微调)、B 类(主线:确定性卡片+全栈 rerank,embedding 不参与)。
- **用正确 en embedding 重跑 A 类**(判分无关 Recall@K):**方向不翻转**(en 下卡片 0.929>代码 0.661),但 **"相似度 0.42<<依赖" 的差距幅度被错语言 embedding 夸大了**(相似度臂被低估);**修正幅度,方向保留**。
- **意外发现(反而强化主线)**:zh 下纯语义 embedding 层,简洁结构卡片一度输给原始代码(0.32<0.46)——**卡片优势靠 BM25+rerank 实现、非纯语义相似**;与主线"确定性+rerank 主导、embedding 次要"一致。
- **B 类主线经核查不受影响**:全栈即便用最差 embedding 也 R@5 0.79(M65),卡片检索是确定性+BM25 主导、rerank 兜底,**0.94 等主线分数不依赖 zh embedding**。
- **净**:无方向翻转,一处幅度修正 + 一个机理澄清;地基稳。**这是第 15 次有据修正,且是"自查地基的次要选择对历史结论的污染范围"——主动查、如实修、给评委佐证主线不受影响。**

## 〔强模型 + 迁移〕界定最致命未控变量 + 破"深而窄"(夯实轮,非修正、是加固/边界)
- **强模型验证(评委第二条,最致命未控变量)**:头号定性"模型侧边际/检索受限"此前生成器全是 14B。用前沿 **deepseek-v4-pro** + **qwen3-235b(比 14B 大 ~17×)** 双口径(det + LLM-judge glm-5,gen≠judge)重验:**前沿 baseline 不高于 14B(det 0.21 vs 0.287,judge deepseek 0.04)、RAG 杠杆更大(4.6×~4.8×)、PE 仍边际(±0.06)**——**"容量受限伪影"被证伪,头号定性加固**。这是把"撑着头号结论的最大未控变量"正面控住,不是修正。`07_泛化夯实/STRONG_MODEL_VALIDATION.md`。
- **迁移验证(评委第三条,"深而窄")**:在第二个任务(调用图,小样本 demo n=30,judge-independent gold=repo_parser)验规律迁移:**喂结构 0.824→1.000(方向迁移),但 baseline 已高(0.82)因调用名在函数源码里可见**——**诚实精炼为"喂结构跨任务有效、其价值在结构最难从局部看出处最大",不夸大为完整泛化**。`07_泛化夯实/TRANSFER_CALLGRAPH.md`。
- **诚实定位**:两项都"两种结果都接受"地诚实记录——强模型本可能翻转(没翻、加固),迁移本可能完整泛化(没有、是方向迁移+幅度任务依赖)。不挑结果。

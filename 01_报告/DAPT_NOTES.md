# DAPT_NOTES.md — 领域自适应预训练记录(M15)

> 项目此前从未探索的方向。本文记录概念区分、轻量化流程、隔离、灾难性遗忘监控与结论。
> **核心预期(待数据验证)**:DAPT 在依赖问答上仍无显著正提升——这将是最深刻的论断佐证:
> **依赖关系不在代码文本的统计分布里,让模型大量"阅读"领域代码也学不会它;依赖是"算"出来的结构事实,不是"读"出来的统计规律。**

## 1. 概念区分(避免与之前的 SFT 混淆)

| | SFT(此前 M5/E40 做的) | DAPT(M15 新方向) |
|---|---|---|
| 数据 | "问题→标准答案"对(538 条) | 纯领域代码,无标签(722 段) |
| 目标 | 教模型**答依赖题**(监督) | 让模型**沉浸领域分布**(next-token 自监督) |
| 性质 | 有监督微调 | 无监督继续预训练 |
| 学到什么 | 答题格式 + 部分依赖模式 | 领域代码的统计规律/风格 |

**关键问题**:让模型"读"大量同领域 Python 库代码,能不能学会"谁 import 谁"这类**跨文件结构事实**?

## 2. 轻量化红线(严守,否则烧数天 + 灾难性遗忘失控)

- **不做完整大规模 DAPT**;最小可行版:LoRA-based 继续预训练 + **硬步数上限 MAX_STEPS=200**。够验证方向即停。
- **必须监控灾难性遗忘**:DAPT 前后都测 12 题通用代码探针 pass@1(`data/code_probe.jsonl`,HumanEval 风格,与依赖问答无关)。
  文献(FSE)显示完整 DAPT 后 HumanEval 可掉 ~24%;本轻量版预期遗忘较小,但仍如实记录。
- 时间/显存设硬上限,接近即停,产出"趋势性结论"而非"完整 DAPT"。

## 3. 隔离(红线,比 SFT 更隐蔽)

- DAPT 语料来自 **click / jinja2 / requests**(`scripts/build_dapt_corpus.py`),共 722 段。
- **排除 flask**:flask 是建在 werkzeug 之上的框架,代码大量 import/引用 werkzeug API,会泄漏 werkzeug 结构。
  代价:DAPT 域少了最像 werkzeug 的 flask,但**隔离绝对干净**(诚实记录这个取舍)。
- **段级过滤**:任何含 "werkzeug" 字样的段一律剔除。最终语料含 werkzeug 段数 = **0**(已核验)。
- 训练脚本启动时再次 assert 语料零 werkzeug(`train_dapt.py`)。

## 4. 流程与结果

- 流程:`build_dapt_corpus.py` → `train_dapt.py`(LoRA 继续预训练 200 步 + 合并)→ 服务 DAPT 模型 → 核心评测 + 遗忘探针。
- 对比:base / DAPT 后(/ 可选 DAPT 后再 SFT)在 werkzeug 依赖问答上的表现 + 通用代码 pass@1 前后。

## 5. 结论(n=15 base 裁判 + 遗忘探针,E43)

| 指标 | base | DAPT 后 | 判定 |
|---|---|---|---|
| 依赖问答 FT-only(无 RAG) | 0.185 | 0.173±0.011 | **无增益**(Δ=−0.012,p=0.099 噪声内) |
| 通用代码 pass@1(遗忘) | 12/12 | 12/12 | **无可测遗忘**(本轻量版) |

- **DAPT 在依赖问答上无正提升**——这正是预期中最深刻的论断佐证:
  **依赖关系不在代码文本的统计分布里,让模型大量"阅读"领域代码也学不会它;
  依赖是"算"出来的结构事实(谁 import 谁,由静态分析确定),不是"读"出来的统计规律。**
- **遗忘代价**:本最小可行版(LoRA 200 步)未见可测遗忘(探针前后均满分)。
  **诚实边界**:① 探针偏易(12 题简单函数,有天花板效应),可能掩盖细微退化;
  ② 文献(FSE)报告**完整大规模 DAPT** 后 HumanEval 可掉 ~24%——本轻量版步数太少故遗忘不明显;
  ③ 未充分探索大规模 DAPT 的"无增益 + 重遗忘"双输区,留作 NEXT_STEPS。
- **性价比裁决**:DAPT 在本任务上无增益(且大规模做还会遗忘),**不进方案**——
  与"微调生成器/检索器都边际"一道,焊死"模型侧努力穷尽边际"的结论。

# DIRTY_SCENARIO.md — 脏场景退化实验(M29:把"定义性碾压"变成"真实价值曲线")

> 第三轮评委最重一刀:核心论断有"定义性真理"软肋——评测测的是"模型能否读懂给定的正确卡片",
> 不是"系统能否在真实脏代码上算准依赖"。本实验直面它:在**静态难算**的依赖上,卡片会算错/漏,测方法退化。

## 1. 脏场景用例(12 条,真实 CPython Lib,非人造极端)
- **动态依赖(3)**:`__import__(变量)`(pickle 按运行时对象的模块名导入)、`importlib.import_module(字符串)`(pydoc/inspect)——
  静态分析**完全算不出目标**,卡片漏掉。
- **条件/可选依赖(9)**:`try: import _wmi except ImportError: _wmi=None`(platform 等)——静态卡片把它列为普通依赖,**不标可选性**。
- gold 由**人工读源码核验**真实行为;full=静态卡片(对这些场景错/缺),baseline=无卡片。

## 2. 结果(独立 Coder 裁判,n=12)
| 场景 | baseline | full(静态卡片) | full−baseline |
|---|---|---|---|
| **对照 clean(M21 CPython)** | 0.05 | 0.94 | **+0.89** |
| **脏场景全体** | 0.531 | 0.083 | **−0.447 (p=0.005)** |
| ├ 条件依赖(9) | 0.42 | 0.11 | −0.31 |
| └ 动态依赖(3) | 0.87 | 0.00 | **−0.87** |
internLM 一致:脏场景 full−base **−0.381(p=0.016)**。

## 3. 裁定:结构化表示的价值随静态分析准确性**翻转符号**
- 干净依赖:full **+0.89**(结构化碾压)。脏依赖:full **−0.45**(结构化**反而有害**)。
- **机理**:静态卡片漏掉动态依赖/不标条件性,而 prompt 要求"只信卡片"——模型被错卡片**带偏**,
  反而不如凭自身对 CPython 的参数知识作答(baseline 0.53 > full 0.08)。**错卡片比没卡片更糟。**
- **这把"前提条件"从声明变成实测曲线**:结构化表示的价值 = f(静态分析准确性)——
  对时数量级正杠杆,错/漏时数量级负杠杆。**不是定义性真理,是有真实边界的价值。**

## 4. 对甲方(RepoMind)的真实含义
- 真实生产代码有大量动态/条件/反射依赖(插件、可选后端、运行时绑定)——**静态方法在这些地方会伤,不是帮**。
- 所以**第一要务是静态分析的覆盖率 + 不确定性检测**:能算准的喂卡片(+0.89),算不准的**宁可不喂也别喂错卡片**(否则 −0.45)。
- 工程建议:给依赖卡片标"置信度/是否动态/是否条件",低置信时不强制模型"只信卡片",留给模型参数知识兜底。

## 5. M30 深探:翻负稳定、有曲线、且 prompt 可救(E56-58)
扩样本22条+3条件验证(自我警惕:不预设翻负成立)。
| 条件(Coder,n=22) | baseline | strict(盲信卡片) | humble(承认可能不全) |
|---|---|---|---|
| 全体 | 0.457 | 0.023 (Δ−0.434***) | 0.866 (救+0.843***) |
| 条件依赖(14) | 0.28 | 0.01 | 0.80 |
| 动态依赖(6) | 0.70 | 0.07 | 0.97 |
| 反射(2) | 1.00 | 0.00 | 1.00 |
**① 翻负稳定**(扩样本仍−0.43);**② 脏度曲线**(反射−1.0>动态−0.63>条件−0.27,漏得越彻底伤越狠);
**③ 可救**:humble prompt完全救回(+0.84)。**翻负是"盲信卡片"prompt缺陷,非本质。**
## 6. 完整价值曲线(最终)
| 场景 | full−baseline |
|---|---|
| 干净依赖(静态算对) | **+0.89**(数量级正杠杆) |
| 脏依赖 + 盲信卡片prompt | **−0.45**(错卡片误导,负杠杆) |
| 脏依赖 + humble prompt | **+0.41**(卡片正确部分帮忙、模型补缺口) |
→ **价值 = f(静态分析准确性, prompt设计)**。结构化表示要配"承认不完整"的prompt才在脏场景稳健。

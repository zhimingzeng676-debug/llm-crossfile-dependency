## 一、旧洞逐条裁决(回数据)

### 洞① 新工作去循环 →【真补，但头条最弱、且避开了硬骨头】
**真的部分**:
- **C(lua)= gcc -H 真编译器、Java(gson)= javalang 真解析器**——这两个是货真价实的独立解析器,
  独立 vs 原 gold 一致 95.8%/100%,去循环 RAG 幅度 0.964/0.538,**循环贡献仅 0~3.6%。这两语言堵死了。**

**没补干净的部分(扣分)**:
1. **工业级头条(CPython)的"去循环"不是真独立解析器**:bundle note 写明 cpy/go 用的是
   **"独立 raw-line 抽取+人工"**——这跟原方法都是正则/逐行扫,**不是 gcc/javalang 那种异构独立途径**。
   而 CPython 恰是最重要的工业级头条,**它的去循环最弱**(一致率 89.5% 最低、循环贡献 10.5% 最高)。
   他们其实演示过 gcc -H 能跑 CPython 的 _json.c,却因"配置脆弱"没用在头条上——**最强的主张配了最弱的解耦**。
2. **去循环子集小且只测"容易"的依赖类型**:lua 7 条(仅 forward)、go 8 条(仅 symbol)。
   去循环只覆盖了 forward/symbol 这类**答案=字面 import 行、本就无歧义**的依赖——两方法当然一致。
   而**真正可能藏循环的硬依赖(传递闭包/别名/跨编译单元)根本没进去循环子集**。
   即:**在最不需要去循环的地方做了去循环,在最需要的地方没做。**
**裁决**:C/Java 堵死;CPython 头条与硬依赖类型**仍是"含 oracle 上界"**,只是循环贡献被旁证压在 ~10%。**部分真补,非干净。**

### 洞② IRR →【表述真改对,但补的旁证是"新的相邻指标"】
**真的部分**:全项目 κ=0.806 **已改标为"两 LLM 裁判间一致性(非人工 IRR)"**(grep 无残留冒充);
**诚实降级**"人工 IRR 未建立(单标注 n=20)=已知局限"——这条态度到位,认了。

**搪塞嫌疑(扣分)**:补的"裁判 vs 确定性 keyword 真值 κ=0.941"作为"客观旁证"——**这又是一次相邻指标**:
- keyword_all 正是**原 eval 用的同一套确定性 gold 来源**,**不独立于评测管线**;
- "LLM 裁判 ≈ keyword 匹配器"κ=0.941,**说明裁判和粗糙的关键词匹配一样粗**——
  这恰恰**反证**不了"裁判能分辨 0.02 级细差"(项目别处曾依赖的),只能证"在 0/1 粗判上一致"。
**裁决**:表述诚实修正(真补);但 κ=0.941 旁证**站不住当"裁判可靠性的客观验证"**——它同源且只验粗判。
人工 IRR 仍是真空,项目方诚实标了局限——**认局限是优秀的,拿同源旁证包装不是。**

### 洞③ 措辞校准 →【真做】
"通杀→在所测项目上一致成立"、"工业级"加 40 用例限定、"跨语言通用→跨所测三语言"——已降级,grep 无残留。✅

## 二、往深挖:前两轮没触及的三层弱点(这才是升优秀的拦路虎)

**深层①(最致命):核心论断部分是"定义性真理"。** 评测测的是"模型能否**读**给定的正确卡片",
不是"系统能否在**真实脏代码上算出/检索到**正确的依赖"。+0.9 = 给答案 vs 不给答案。
**RAG 数量级碾压**几乎是同义反复——把算好的依赖喂进去,当然赢过不喂。真正难的(在大规模脏代码上把依赖**算准算全**)
被**假设**掉了,只在 40 条干净用例上验过检索可行性。**对甲方真正的风险点(私有代码依赖能否被静态分析覆盖)恰恰没测。**

**深层②:baseline 近零部分是"问题格式"的产物,不全是"模型不行"。** 问题要"列出全部精确 import 文件",
**连人类专家也无法凭记忆列全** CPython 某文件的精确 import。所以"模型侧边际"部分是
**"精确召回格式打败了参数知识"**,不全是"模型不能推理依赖"。换成"判断 A 是否依赖 B"的是非题,baseline 可能没那么惨。

**深层③:实际只有一个能用的裁判(Coder)。** internLM 全程系统性宽松/饱和(Java −0.016、indirect 全 1.0、知名库给免费分),
每次都"以 Coder 为准"。**"两独立家族裁判"名不副实**——一个长期失效。**判分独立性实质压在 Coder 一个模型上。**

## 三、到"优秀"还差什么(明确门槛)
1. **头条去循环升级到真解析器**:用 gcc/clang 解决 CPython 头条的 #include 解耦(已证可跑),别用 raw-line。
2. **造一个真正独立于评测管线的金标准**:非 keyword、非 LLM、人工核验的小子集(哪怕 15 条),覆盖**硬依赖类型**(传递/别名),在其上复测——这才真去循环。
3. **补一个真·第二人类标注者**(或明确承认做不到,把"人工 IRR 真空"写进交付摘要的显眼处,而非用 κ=0.941 软化)。
4. **回应"定义性"批评**:设计一个"依赖结构难静态计算"的脏场景(动态 import/宏/条件依赖),测方法在那里退化多少——把"前提条件"从声明变成实测边界。
5. **判分独立性**:换一个真正会跟 Coder 分歧的强裁判(如 GPT-4 级或第三家族 14B+),而非长期失效的 internLM。

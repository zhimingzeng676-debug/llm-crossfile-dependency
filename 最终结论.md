## 一、硬结论(主导杠杆-大效应〔M56:非"数量级"〕、跨多维度稳健、有 LLM-judge-independent 客观锚)
1. **结构化依赖表示是主导杠杆(大效应:严打约 4-5×/宽松约 2×;〔M56〕非"数量级")**:baseline 0.05~0.19 → 结构化依赖卡片 0.77~0.94。
   - 稳健维度:两 LLM 裁判 + 56 用例配对 + 检索指标(Recall/MRR)+ keyword 客观真值(judge vs 真值 κ=0.94)。
   - 跨语言:Go +0.93 / Java +0.60 / C +0.86(p<1e-6,E47);跨规模:CPython 工业级 +0.89(E48);腾讯 rapidjson +0.83(E52)。
   - **去循环验证**:werkzeug ast 独立重抽(与 tree-sitter 100% 吻合,E45);新工作 gcc -H/javalang 真解析器(lua/gson 循环贡献 0~3.6%,E53)。
2. **模型侧优化一律边际/无效**(对"注入依赖知识"):
   - PE/CoT 经三独立裁判全部不显著(E44,原 +0.02 系 self-judge+伪重复伪影,**已收回**)。
   - 微调:生成器 +0.04 冗余(E40)、检索器 −0.02 过度专化(E41)、DoRA/IA³/DAPT 边际/无(E42-43)、RLCoder 式 RL 检索器 0.958<静态 1.000(E51,**学习式检索器≤静态分析,穷尽封口**)。
   - 机理:依赖是"算"出来的结构事实,不是"读/猜"出来的统计规律——计算它的静态分析是 oracle。
3. **检索可行性在工业级仍成立**:1456 卡片池(14×),rerank=80 捕获 97.5% 正确卡片,未成瓶颈(E48)。
4. **图表示选型**:直接依赖 B0(1-hop 卡片)即够、符号查询需 B0(含节点属性);**完整图结构子图检索 B3 是过度工程**(B3 0.69<B2 0.90,E49)。价值在"物化什么结构"非"图检索多复杂"。

## 二、方向性发现(证据偏薄,不可当硬结论,经第四轮评委+M33-34反复打磨后的诚实状态)
5. **脏依赖场景:结构化表示的价值取决于静态分析准确性 + prompt 设计**:
   - **判分无关客观指标证实(稳健)**:humble prompt 让模型正确识别脏依赖(动态/条件)的比例 strict 21% → humble **100%**(E63)——humble 是实质改进,非风格假象。
   - **戏剧性端到端幅度(下修)**:M29-31 的"翻负 −0.45 / 救回 +0.84"端到端分数幅度,经中性措辞复测(E60)发现**部分是问题措辞 artifact + 裁判放大**——中性措辞下 strict−base 仅 +0.10(ns)。
   - **可执行建议(朴素版)**:依赖卡片由静态分析生成、对动态/条件依赖会不全,prompt 应承认这点、别让模型盲信卡片当全集。量化收益是方向性的。

## 三、已收回(诚实推翻的旧结论)
- **CoT/PE 增益**(原 +0.02 p=0.001):self-judge + 伪重复伪影,三独立裁判全不显著,**收回**(E44)。
- **"FT 砸间接 −0.20"**:5 样本假象,n=18 未复现,**推翻**(早期 M8)。
- **戏剧性脏价值曲线 +0.89/−0.45/+0.41**:端到端幅度部分是措辞 artifact,**下修**为方向性发现(E60)。

## 四、已知局限(摆显眼处,不用相邻指标包装,见 HONEST_LIMITATIONS.md)
1. **人工标注者间 IRR 真空**:校准仅单标注者 n=20;**撤回**了"keyword 真值 κ=0.941"同源旁证(它与 eval 同源、不独立)。
2. **判分独立性实质靠 Coder 一个裁判**:internLM 系统性宽松/饱和、脏场景顺从偏好(M34/35)；Coder 相对干净(无长度/hedging 偏好)但有轻微自信错答偏好(0.25,E64)。
3. **baseline 近零部分是"精确召回格式"产物**,非"模型完全不能推理"。
4. **脏/humble 线证据偏薄**:小规模、单/少项目、工业脏代码外推未验证。
5. **评级利益冲突**:第三四轮自评不可作独立认证。

## 五、给 RepoMind 的最终建议(决策语言,见 VALUE_FOR_REPOMIND.md)
**资源优先级**:依赖结构静态分析的质量+覆盖 + 检索深度(主导杠杆/大效应,〔M56〕非"数量级")>> 更大模型/复杂 prompt/微调/完整图 RAG(边际)。
**关键工程铁律**:① 把钱投在"静态分析算准算全"(别名/传递闭包/跨语言可抽取边)；② 多跳依赖文本化物化路径即可,别上图数据库 RAG；③ **prompt 别让模型盲信依赖卡片**(对动态/条件依赖留模型判断)；④ 评测必须 gen≠judge + 配对统计 + 金标准解耦。
**适用边界**:依赖结构**能静态计算**时大幅碾压(主导杠杆,〔M56〕非"数量级");**算不出时**(宏/动态加载/隐式链接/运行时绑定)方法退化甚至有害。
